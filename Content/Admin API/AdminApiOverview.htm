HyperStore Admin API

Cloudian HyperStore provides a RESTful HTTP API through which you can provision users and groups, manage rating plans and quality of service (QoS) controls, and perform other administrative tasks. This Admin API is supported by the HyperStore Admin Service, which runs on the same nodes as your HyperStore S3 Service. By default the HTTP listening port for the Admin API is 18081.

The Cloudian Management Console (CMC) accesses the Admin API to implement its provisioning and reporting functions. You also have the option of accessing the Admin API directly, using a command line tool such as cURL or a REST client application of your own creation.

HyperStore Admin API response payloads are JSON encoded. For POST or PUT requests that require a request payload, the request payloads must be JSON encoded as well.

The HyperStore Admin API is not designed to be directly exposed to end users of the Cloudian HyperStore storage service. It is intended for HyperStore system operators to use from within an internal network. Do not directly expose the Admin Service to an external network.

Admin API Behavior in Multi-Region Systems

If your HyperStore system has multiple service regions, then:

  • For the Admin Service in the default service region, all of the Admin API operations in this document are supported.
  • For the Admin Service in regions other than the default region, only the following sub-set of Admin API operations is allowed:
    • POST Usage Data for Active Users
    • POST Usage Data for All Users
    • POST Usage Data Rollup
    • POST Usage Data Repair for Active Users
    • GET System Version

If in a non-default region you send your local Admin Service a request for an operation other than those listed above, you will receive a 403:Forbidden response.

If your HyperStore S3 service has only one service region, then your Admin Service supports all of the Admin API operations.
