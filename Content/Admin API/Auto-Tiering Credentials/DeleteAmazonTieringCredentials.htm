DELETE Amazon Tiering Credentials

[Admin API Method]

Use this Admin API method to delete Amazon S3 access credentials that were previously stored to Redis via the POST Amazon Tiering Credentials method.

Request Format

Syntax

DELETE /tiering/credentials[?bucketName=xxx]

Query Parameters

bucketName

(Optional, string) HyperStore S3 bucket for which to delete previously supplied Amazon S3 credentials. If no bucketName is supplied, the request deletes the previously supplied system default access credentials.

Request Payload

None.

 Response Format

Response Status

This method returns only the Common Status Codes.

Response Payload

None.

 Example

In this example, the Amazon tiering credentials for the HyperStore bucket named "bucket1" are successfully deleted.

# Request ->

DELETE http://192.168.2.16:18081/tiering/credentials?bucketName=bucket1

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Mon, 23 Nov 2015 12:15:50 GMT
