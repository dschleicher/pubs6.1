GET Amazon Tiering Credentials

[Admin API Method]

Use this Admin API method to retrieve an Amazon S3 access key that was previously stored to Redis via the POST Amazon Tiering Credentials method.

This operation retrieves only the access key — not the corresponding secret key.

Request Format

Syntax

GET /tiering/credentials[?bucketName=xxx]

Query Parameters

bucketName

(Optional, string)   HyperStore S3 bucket for which to retrieve a previously supplied Amazon S3 access key, used for auto-tiering to Amazon. If no bucketName is supplied, the request retrieves a previously supplied system default access key.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
204 No Tiering Credentials found.

Response Payload

None.

 Example

In this example, the Amazon S3 access key that is used for auto-tiering from the HyperStore bucket named "bucket1" is sucessfully retrieved.

# Request ->

GET http://192.168.2.16:18081/tiering/credentials?bucketName=bucket1

# Response ->

Status Code: 200 OK
Content-Length: 20
Content-Type: text/plain
Date: Mon, 23 Nov 2015 12:00:43 GMT
