POST Amazon Tiering Credentials

[Admin API Method]

Use this Admin API method to upload default corporate AWS credentials or bucket-specific AWS credentials in support of the HyperStore auto-tiering feature.

Request Format

Syntax

POST /tiering/credentials?accessKey=xxx&secretKey=xxx[&bucketName=xxx]

Query Parameters

accessKey

(Mandatory, string) Access key for the Amazon S3 account that you want to use as your auto-tiering destination.

 secretKey

(Mandatory, string) Secret key for the Amazon S3 account that you want to use as your auto-tiering destination.

 bucketName

(Optional, string) Name of the HyperStore S3 bucket for which to use the specified accessKey and secretKey when auto-tiering to Amazon. If no bucketName is specified, then the accessKey and secretKey are used as the system default credentials for auto-tiering to Amazon.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
204 Bucket does not exist.
400 Missing required attributes : {accessKey, secretKey}

Response Payload

None.

 Example

In this first example, corporate default AWS credentials are successfully posted to the system.

# Request ->

POST http://192.168.2.16:18081/tiering/credentials?accessKey=0f6c623e5957a7a370a0
&secretKey=BLeg7PcK7VN38VfX7ltBix4Sp5RhTocrfw8Db+2R

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Mon, 23 Nov 2015 11:44:49 GMT
