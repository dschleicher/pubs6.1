GET User's Bill

[Admin API Method]

Use this Admin API method to retrieve a user’s bill, after it’s been generated. (To generate the bill, use the POST User's Bill method.)

You can retrieve bills only for a completed month. You cannot retrieve a bill for the current, in-progress month.

Request Format

Syntax

GET /billing?[userId=xxx&groupId=xxx][canonicalUserId=xxx]&billingPeriod=xxx

Query Parameters

userId

(Optional, string)  User ID for the user whose bill is being retrieved.

Do not use the "userId" and "groupId" parameters for a user who has been deleted from the S3 service. For a deleted user, use the "canonicalUserId" parameter instead.

You must either use "userId" together with "groupId", or use "canonicalUserId" by itself.

 groupId

(Optional, string) Group ID for the user whose bill is being retrieved.

You must either use "userId" together with "groupId", or use "canonicalUserId" by itself.

 canonicalUserId

(Optional, string) System-generated canonical ID for the user whose bill is being retrieved. Use this parameter if you want to retrieve a bill for a user who has been deleted from the S3 service. If you don’t know the user’s canonical ID, you can obtain it by using the GET User List method (this method can retrieve user profile information — including canonical ID — for all deleted users within a specified group).

You must either use "userId" together with "groupId", or use "canonicalUserId" by itself.

 billingPeriod

(Mandatory, string) Specifies year and month of bill to retrieve. The system uses GMT time when determining the beginning and end of a month. Format is yyyyMM.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
204 Billing data does not exist
400 User does not exist
400 Missing required parameter : {billingPeriod}
400 Conflicting parameters: {canonicalUserId, groupId, userId}

Response Payload

Bill JSON object.

 Example

In this example, an existing bill for a user named "test" is successfully retrieved. The bill is for October, 2015, and indicates that total charges incurred by that user are $10.94, based on the user’s storage volume having been at an average level of 108GBs during the month of October.

# Request ->

GET http://10.10.3.26:18081/billing?userId=test&groupId=test&billingPeriod=201510
