MonitorSystemInfo

[Admin API Object]

The MonitorSystemInfo object is returned as the response payload for this Admin API operation:

  • GET Monitoring Data for Region

The MonitorSystemInfo object consists of the attributes below.

Within the MonitorSystemInfo object, all statistics of type MonitorStat are formatted as
"<statName>":{"timestamp":"<UTCMilliseconds>","value":"<statValue>"}.

diskAvailKb

(MonitorStat) Across the whole service region, the total mounted disk space remaining still available for Cassandra data directory or HyperStore data directory storage. A disk’s "reserved-blocks-percentage" (the portion of the disk that’s reserved for privileged processes) does not count as available space.

 diskTotalKb

(MonitorStat) Across the whole service region, the total size of the disks mounted for Cassandra data directories or HyperStore data directories (the HyperStore File System [HSFS] and erasure coding file system). This stat does not include other any other disks on your HyperStore hosts — such as those that are being used only for the OS or application files, for example.

 nodeStatuses

(List<NodeStatus>) List of NodeStatus objects, one for each node in the service region.

 s3GetLatency

(MonitorStat) Across the whole service region, the average latency for completing S3 GET transactions, in milliseconds. This stat is recalculated each five minutes, based on the most recent five minutes of transaction data. HEAD transactions are counted toward this stat also.

 s3GetThruput

(MonitorStat) Across the whole service region, the data throughput for S3 GET transactions, expressed as MB per second. This stat is recalculated each five minutes, based on the most recent five minutes of transaction data. HEAD transactions are counted toward this stat also.

 s3GetTPS

(MonitorStat) Across the whole service region, the number of S3 GET transactions processed per second. This stat is recalculated each five minutes, based on the most recent five minutes of transaction data. HEAD transactions are counted toward this stat also.

 s3PutLatency

(MonitorStat) Across the whole service region, the average latency for completing S3 PUT transactions, in milliseconds. This stat is recalculated each five minutes, based on the most recent five minutes of transaction data. POST transactions are counted toward this stat also.

 s3PutThruput

(MonitorStat) Across the whole service region, the data throughput for S3 PUT transactions, expressed as MB per second. This stat is recalculated each five minutes, based on the most recent five minutes of transaction data. POST transactions are counted toward this stat also.

 s3PutTPS

(MonitorStat) Across the whole service region, the number of S3 PUT transactions processed per second. This stat is recalculated each five minutes, based on the most recent five minutes of transaction data. POST transactions are counted toward this stat also.

 status

(ServiceStatus) High-level service status for the system as a whole. Formatted as {"ipaddr":"<empty>","status":[<one of "OK" or "SVC_DOWN">],"value": "<sameAsStatus>","timestamp":"<UTCMilliseconds>"}.

The "ipaddr" value will be empty or null.

The "status" will be formatted as a list but with just one member string. Status string meanings are:

  • "OK" — All HyperStore services are up and running on all nodes in the service region.
  • "SVC_DOWN" — A HyperStore service (Admin, Cassandra, HyperStore, Redis QoS, Redis Credentials, Redis Monitor, or S3) is down on one or more nodes in the service region.

The "value" attribute will be identical to the "status" attribute, except formatted as a string rather than as a list. For example:

"status":["SVC_DOWN"],"value":"[SVC_DOWN]"
