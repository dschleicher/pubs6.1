RuleClass

[Admin API Object]

A list of RuleClass objects is nested within the RatingPlan object.

The RuleClass object consists of the following attributes:

ruleclassType

(string) Type of rating rule: one of {STORAGE_BYTE, BYTES_IN, BYTES_OUT, HTTP_GET, HTTP_PUT, HTTP_DELETE}. These are the service usage dimensions for which pricing can be set.

 rules

(List<Pair>) List of rating rules to apply to the rule class type. There is one rule (one Pair object) per pricing tier.

Example

This is an example of a RuleClass object. It indicates a two-tier pricing structure for HTTP GETs (this RuleClass object contains two Pair objects, one per tier). The first 100,000 GETs are priced at .02 currency units per 10,000 GETs. Beyond 100,000, GETs are priced at .01 currency units per 10,000 GETs. The currency — such as U.S dollars or Japanese yen — would be specified in the parent RatingPlan object.

{"rules":[{"second":"0.02","first":"10"},{"second":"0.01","first":"0"}],"ruleclassType":"HTTP_GET"}

For detail on how per-tier price rules are set — including information about the units by which different service usage dimensions (such as storage bytes or HTTP GET requests) are measured for pricing purposes — see the documentation of the Pair object.
