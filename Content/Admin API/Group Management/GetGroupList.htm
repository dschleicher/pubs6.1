GET Group List

[Admin API Method]

Use this Admin API method to retrieve a list of user groups in your HyperStore system. For each group the group’s profile is returned. You can optionally filter the search by group ID prefix.

This method supports limiting the number of groups that are returned for each request/response pair, if there are many user groups in your system. It also supports use of an offset value. With the limiter and the offset, you can use multiple sequential requests in order to paginate a large result set.

Request Format

Syntax

GET /group/list[?prefix=xxx][&limit=xxx][&offset=xxx]

Query Parameters

prefix

(Optional, string)  If specified, a group ID prefix to use for filtering. For example, if you specify "prefix=usa" then only groups whose group ID starts with "usa" would be retrieved.

Defaults to empty string (meaning that no prefix-based filtering is performed).

 limit

(Optional, integer) For purposes of pagination, the maximum number of groups to return in one response. If more than this many groups meet the filtering criteria, then the actual number of groups returned will be "limit plus 1". The last group returned — the "plus 1" — is an indicator that there are more matching groups than could be returned in the current response (given the specified "limit" value). That group’s ID can be used as the "offset" value in a subsequent request. Note that if the offset group happens to be the last group in the entire set of matching groups, the subsequent query using the offset will return no groups.

Defaults to 100.

 offset

(Optional, string) Group ID with which to start the response list of groups, sorted alphanumerically. The "offset" parameter can be used for purposes of pagination within a large result set that is being retrieved via multiple requests. See the description of "limit" above for more information.

If "offset" is not specified, the first group in the response list will be the alphanumerically first group from the entire result set.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Limit should be greater than zero

Response Payload

List<GroupInfo> in JSON.

 Example

In this example, profiles for all the user groups in the system are successfully retrieved. There are only four groups in the system. The set of profiles in the response body is sorted alphanumerically by groupId value.

# Request ->

GET http://192.168.2.16:18081/group/list

# Response ->

Status Code: 200 OK
Content-Length: 266
Content-Type: application/json
Date: Wed, 23 Sep 2015 18:45:13 GMT
[{"active":"true","groupName":"Development","groupId":"Dev","ldapGroup":null},
{"active":"true","groupName":"Technical Publications","groupId":"Pubs","ldapGroup":null},
{"active":"true","groupName":"Quality Assurance","groupId":"QA","ldapGroup":null},
{"active":"true","groupName":"Sales & Marketing","groupId":"Sales","ldapGroup":null}]

For more information about GroupInfo object attributes, see GroupInfo.
