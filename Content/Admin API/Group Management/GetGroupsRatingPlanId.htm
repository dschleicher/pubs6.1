GET Group's Rating Plan ID

[Admin API Method]

Use this Admin API method to retrieve a group’s assigned rating plan ID. This method does not retrieve the whole rating plan object — only the rating plan ID.

If you want to subsequently retrieve the whole rating plan associated with a given rating plan ID, you can do so by using the GET Rating Plan method.

Request Format

Syntax

GET /group/ratingPlanId?groupId=xxx[&region=xxx]

Query Parameters

groupId

(Mandatory, string) Group ID whose rating plan ID is being retrieved.

 region

(Optional, string) Service region with which the rating plan is associated. If not specified, the default region is assumed.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
204 Rating Plan does not exist
400 Missing Required parameters : {groupId}
400 Region {region} is not valid

Response Payload

string (RatingPlanId)

 Example

In this example the "Dev" group’s rating plan ID is successfully retrieved.

# Request ->

GET http://192.168.2.16:18081/group/ratingPlanId?groupId=Dev

# Response ->

Status Code: 200 OK
Content-Length: 10
Content-Type: text/plain
Date: Wed, 23 Sep 2015 19:24:21 GMT
Default-RP
