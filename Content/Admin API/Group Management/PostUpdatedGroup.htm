POST Updated Group

[Admin API Method]

Use this Admin API method to change an existing group’s name or its service status. These are the only attributes that can be changed through this method.

If you suspend a user group by changing the group’s status to inactive, none of the users in the group will be able to access the S3 storage system or log into the CMC.

To change a group’s rating plan, use the POST Group's Rating Plan ID method. To change a group’s quality of service settings, use the POST QoS Limits method.

Request Format

Syntax

POST /group

Query Parameters

None.

Request Payload

GroupInfo JSON object.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Group does not exist
400 Missing required attribute : {groupId}
400 Invalid Active Status for Post Group
400 Invalid JSON Object

Response Payload

None.

 Example

In this example, the Human Resources group’s status is successfully changed to inactive. Now no members of this group will be able to log into the CMC or access the HyperStore S3 storage system.

# Request ->

POST http://192.168.2.16:18081/group
Content-Type: application/json
{"active":"false","groupName":"Human Resources","groupId":"HR"}

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 23 Sep 2015 19:14:22 GMT

For more information about GroupInfo object attributes, see GroupInfo.
