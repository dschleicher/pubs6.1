DELETE QoS Limits

[Admin API Method]

Use this Admin API method to:

  • Delete QoS limits that have been assigned to a specific user. If you delete user-specific QoS limits, the system will automatically assign the user the default user-level QoS limits associated with the group to which the user belongs.
  • Delete QoS limits that have been assigned to a specific group. If you delete group-specific QoS limits, the system will automatically assign the group the regional default QoS limits.

Essentially, this method allows you to clear user-specific or group-specific QoS overrides so that default QoS settings are used instead.

Request Format

Syntax

DELETE /qos/limits?userId=xxx&groupId=xxx[&region=xxx]

Query Parameters

userId

(Mandatory, string) User ID of the user whose QoS settings are being deleted. (If you are deleting group-level QoS settings rather than a user-level QoS settings, set the userid to "*". See Note below.)

 groupId

(Mandatory, string) Group ID of the user whose QoS settings are being deleted. (Or, Group ID of the group whose group-level QoS settings are being deleted. See Note below.)

 region

(Optional, string) Service region with which the QoS settings are associated. If not specified, the default region is assumed.

If you want to delete the group-level QoS settings for a group named "Enterprise10", in the default region, use the following syntax:

DELETE /qos/limits?userId=*&groupId=Enterprise10

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing required parameters : {userId, groupId}
400 Region {region} is invalid

Response Payload

None.

 Example

In this example the user-specific QoS settings for user "kwring" are successfully deleted. Now, the default user-level QoS settings assigned to the "Dev" group will be applied to user "kwring". If no user-level QoS settings have been specifically assigned to the "Dev" group, then the default user-level QoS settings for the region will be applied to user "kwring".

# Request ->

DELETE http://192.168.2.16:18081/qos/limits?userId=kwring&groupId=Dev

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 30 Sep 2015 15:17:13 GMT
