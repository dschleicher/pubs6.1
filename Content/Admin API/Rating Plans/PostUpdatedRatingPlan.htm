POST Updated Rating Plan

[Admin API Method]

Use this Admin API method to revise the contents of an existing rating plan.

If you wish you can revise the contents of either of the two pre-packaged rating plans that come bundled with HyperStore: the system default rating plan (plan ID "Default-RP") or the whitelist rating plan (plan ID "Whitelist-RP"). Note though that in the case of "Whitelist-RP", any storage pricing (price per GB-month of storage) that you specify in the plan is ignored by the system. For data storage billing, a user’s regular assigned rating plan pricing is applied even if the user accesses HyperStore exclusively from a whitelisted source IP address. Only the traffic related pricing (data transfer in/out and HTTP requests) in the "Whitelist-RP" is applied by the system.

Request Format

Syntax

POST /ratingPlan

Query Parameters

None

Request Payload

RatingPlan JSON object.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Rating Plan does not exist
400 Missing required parameter : {id, name}
400 Invalid JSON Object

Response Payload

None.

 Example

In this example a rating plan is successfully modified.

# Request ->

POST http://192.168.2.16:18081/ratingPlan
Content-Type: application/json
{"mapRules":{"HG":{"rules":[{"first":"0","second":"0"}],"ruleclassType":"HTTP_GET"},"HD":{"rules":
[{"first":"0","second":"0"}],"ruleclassType":"HTTP_DELETE"},"BI":{"rules":[{"first":"0","second":
"0"}],"ruleclassType":"BYTES_IN"},"BO":{"rules":[{"first":"0","second":"0"}],"ruleclassType":
"BYTES_OUT"},"HP":{"rules":[{"first":"0","second":"0"}],"ruleclassType":"HTTP_PUT"},"SB":
{"rules":[{"first":"1","second":"0"},{"first":"0","second":"3"}],"ruleclassType":"STORAGE_BYTE"}},
"currency":"USD","name":"Silver Level Plan","id":"Silver"}

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 30 Sep 2015 12:28:50 GMT
