DELETE (Disable) Storage Policy

[Admin API Method]

Use this Admin API method to disable a storage policy so that it can no longer be assigned to new buckets. If you disable a policy that is currently used by existing buckets, then those existing buckets can continue to use the policy, but the policy will not be available to use for new buckets.

You cannot disable a policy that is currently the default policy for the region. If you want to disable such a policy, first promote a different policy to be the new default by using either the PUT New Storage Policy method (to make a new policy the default policy) or the POST Updated Storage Policy method (to make an existing policy the default policy). Then you can disable the former default policy.

Request Format

Syntax

DELETE /bppolicy/disable?policyId=xxx

Query Parameters

policyId

(Mandatory, string) System-generated unique identifier of the policy. Note that this is not the policy name (which is set by the administrator who created the policy) but rather the unique policy ID that the system generated upon policy creation.

If you’re not sure what the policy’s policyId is you can retrieve it by using the GET Storage Policy List method. That method returns a list of existing policies and their attributes, including each policy’s policyId.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing policyId
403 Can’t update PENDING policy
403 Can’t disable default policy
404 Policy does not exist

Response Payload

None.

 Example

In this example a storage policy is successfully disabled.

# Request ->

DELETE http://10.50.200.201:18081/bppolicy/disable?policyId=37abc6462910b3f00686e53c96fea380

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Mon, 05 Oct 2015 15:29:54 GMT
