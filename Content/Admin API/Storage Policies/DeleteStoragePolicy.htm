DELETE Storage Policy

[Admin API Method]

Use this Admin API method to delete a storage policy, if the policy is not currently associated with any storage buckets. You cannot delete a policy that is being used by one or more buckets.

If a policy is being used by some existing buckets and you do not want it to be available to new buckets, you can use the DELETE (Disable) Storage Policy method to disable the policy rather than deleting it.

Request Format

Syntax

DELETE /bppolicy?policyId=xxx

Query Parameters

policyId

(Mandatory, string) System-generated unique identifier of the policy. Note that this is not the policy name (which is set by the administrator who created the policy) but rather the unique policy ID that the system generated upon policy creation.

If you’re not sure what the policy’s policyId is you can retrieve it by using the GET Storage Policy List method. That method returns a list of existing policies and their attributes, including each policy’s policyId.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing policyId
403 Can’t update PENDING policy
403 Can’t delete policy
404 Policy does not exist

Response Payload

None.

 Example

In this example a storage policy is successfully deleted.

# Request ->

DELETE http://10.50.200.201:18081/bppolicy?policyId=e810cb2dc3be840d8a07e5dfb2d4d997

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Mon, 05 Oct 2015 14:28:16 GMT
