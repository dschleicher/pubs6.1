PUT New Storage Policy

[Admin API Method]

Use this Admin API method to create a new storage policy.

The system supports a configurable maximum number of policies [XREF] (default = 8). If this many policies already exist, the system will reject a request to create a new policy and will return an HTTP 403 response. Policies with status "Active", "Pending", or "Disabled" count toward the maximum.

Request Format

Syntax

PUT /bppolicy

Query Parameters

None.

Request Payload

BucketProtectionPolicy JSON object, with the policyId attribute present and set to empty ("").

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 No consistency level
400 Invalid consistency level
400 Invalid request. Replica/EC must be specified
400 Invalid replication factor
400 Invalid ec distribution
400 No DC specified for EC
400 Invalid ec distribution. No primacy DC
400 Invalid region
403 Maximum protection policy count reached
403 No default exists and policy created for specific group

Response Payload

BucketProtectionPolicy JSON object, with the policyId attribute given a system-generated value.

 Example

In this example a new storage policy — specifically, an erasure coding policy — is successfully created. Note that in the request, the "policyId" attribute is present and set to empty (""). In the response the system-generated policyId is specified.

# Request ->

PUT http://10.50.200.201:18081/bppolicy
Content-Type: application/json
{"policyName":"ec-policy","policyDescription":"2+1 Erasure Coding","consistencyLevel":{"read":
["LOCAL_QUORUM"],"write":["ALL"],"writeNew":["ALL"],"metadataR":["QUORUM"],"metadataW":["QUORUM"],
"readEC":["LOCAL_QUORUM"],"writeEC":["ALL"]},"ecscheme":{"k":2,"m":1,"dc":["DC1"],"type":"DC"},
"defaultProtectionScheme":"EC","status":"ACTIVE","replicationScheme":null,"systemDefault":false,
"groups":[],"policyId":"","region":"region1"}
