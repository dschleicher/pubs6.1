Admin API Methods for Storage Policies

This group of Admin API methods is for managing HyperStore storage policies (also known as bucket protection policies). Storage policies define a way of protecting stored data by means of replication or erasure coding. There are API methods for creating, updating, retrieving, disabling, and deleting storage policies.

The Cloudian Management Console’s Storage Policies functionality is implemented by calling the methods in this section.

Once created, storage policies can be assigned to S3 storage buckets at the time that the buckets are created, either through the CMC or through HyperStore extensions to the S3 API. For general information about the storage policy feature, see Storage Policies Feature Overview.

  • DELETE Storage Policy
  • GET Storage Policy
  • GET Storage Policy List
  • GET Storage Policy Usage (Bucket List)
  • POST (Re-Enable) Storage Policy
  • POST Updated Storage Policy
  • PUT New Storage Policy
