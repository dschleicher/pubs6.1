DELETE Notification Rule

[Admin API Method]

Use this Admin API method to delete a notification rule.

Request Format

Syntax

DELETE /monitor/notificationrule?ruleId=xxx[?region=xxx]

Query Parameters

ruleId

(Mandatory, string)  ruleId of the NotificationRule object to delete.

If you do not know the ruleId of the rule that you want to delete, you can retrieve it by using the GET Notification Rules method. That method returns a list of rules which includes each rule’s ruleId.

 region

(Optional, string) Service region from which to delete the notifcation rule. If you do not specify a region, the rule will be deleted from your default region.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing required parameters : {ruleId}
400 Notification rule does not exist : {ruleId}

Response Payload

None.

 Example

In this example a notification rule is successfully deleted.

# Request ->

DELETE http://10.50.10.16:18081/monitor/notificationrule?ruleId=182e1aa5-ae2f-4075-ac59-cb39e51cb0d5

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Fri, 04 Dec 2015 14:40:28 GMT
