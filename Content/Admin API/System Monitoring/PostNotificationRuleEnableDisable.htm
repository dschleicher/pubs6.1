POST Notification Rule Enable/Disable

[Admin API Method]

Use this Admin API method to disable one or more existing notification rules, or to enable one or more rules that are currently disabled. When a monitoring rule is disabled, the rule will not be used to trigger system event notifications. A record of the rule remains in the system, however, and you can subsequently re-enable the rule if you wish.

An individual notification rule can also be enabled or disabled by using the POST Updated Notification Rule method and setting the "enabled" attribute to the desired value (true/false) within the NotificationRule object in the request body.

Request Format

Syntax

POST /monitor/notificationruleenable

Query Parameters

None.

Request Payload

NotificationRulesEnable JSON object.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing required parameters : {ruleId}
400 Notification rule does not exist : {ruleId}

Response Payload

None.

 Example

In this example three notification rules are successfully disabled.

# Request ->

POST http://10.50.10.16:18081/monitor/notificationruleenable
Content-Type: application/json
{"regionId":"region1","enable":false,"ruleList":["4f9c4144-c6ad-4756-931c-2489d488b579",
"08484ec1-640d-4c7c-9a33-235c1899fd7c","e2065231-2ae3-44da-a76a-63eb41e9f1ad"]}

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Fri, 19 Feb 2016 02:14:48 GMT
