Admin API Methods for System Monitoring

This group of Admin API methods is for monitoring the health and performance of your HyperStore system. There are methods for retrieving system and node statistics and for implementing system alert (notification) functionality.

The Cloudian Management Console’s Data Centers interface, Node Status interface, Node Activity interface, Alerts interface, and Alert Rules interface are implemented by calling the methods in this section. Note that in the CMC the term "alert" is used rather than "notification" (which is used in Admin API method and object names); for HyperStore these two terms mean the same thing.

  • DELETE Notification Rule
  • GET Event List
  • GET Monitored Node List
  • GET Monitoring Data for Node
  • GET Monitoring Data for Region
  • GET Notification Rules
  • POST Event Acknowledgement
  • POST Notification Rule Enable/Disable
  • POST Updated Notification Rule
  • PUT New Notification Rule
