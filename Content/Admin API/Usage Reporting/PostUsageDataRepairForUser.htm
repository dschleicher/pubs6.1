POST Usage Data Repair for User

[Admin API Method]

This method performs usage data repair for a single specified user. For background information on usage data repair, see Validating Usage Data for Storage Levels.

This operation does not update the group-level usage counters for the group to which the user belongs. For information about doing the latter, see POST Usage Data Repair for Group or System — particularly the "summarizeCountsOnly" option. This is relevant especially when you are repairing multiple individual users within a group, one at a time, using the POST Usage Data Repair for User method. In that case you should subsequently update the group-level usage counters for the group, using the POST Usage Data Repair for Group or System method with the "summarizeCountsOnly" option.

Request Format

Syntax

POST /usage/repair/user?groupId=xxx&userId=xxx[&region=xxx]

Query Parameters

groupId

(Mandatory, string) ID of the group to which the target user belongs.

 userId

(Mandatory, string) ID of the user for whom usage data repair is to be performed.

 region

(Optional, string) Service region for which to perform the repair. If the region parameter is not specified, the repair is performed for all service regions.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing or invalid parameters

Response Payload

None.

 Example

In this example a usage data repair operation is successfully completed for user "cprog".

# Request ->

POST http://192.168.2.16:18081/usage/repair/user?groupId=Dev&userId=cprog

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 30 Sep 2015 14:19:50 GMT
