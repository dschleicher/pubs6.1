Admin API Methods for Usage Reporting

This group of Admin API methods is for managing HyperStore usage reporting. The usage reporting feature captures service usage data for users and groups. There are API methods for aggregating usage data and ensuring its accuracy — many of these methods are invoked regularly by HyperStore system cron jobs. There are also methods for retrieving usage data for specified users or groups.

The Cloudian Management Console’s Usage By Users & Groups functionality is implemented by calling some of the methods in this section.

For an overview of the HyperStore usage reporting feature, see Usage Reporting Feature Overview.

If you want to bill for data upload or download volume, or for HTTP request volume, you must enable the Track/Report Usage for Request Rates and Data Transfer Rates setting in the CMC’s Configuration Settings page. By default this setting is disabled and the system does not maintain per-user HTTP request counts and data transfer byte counts.

  • DELETE Usage Data
  • GET Usage Data
  • POST Usage Data for Active Users
  • POST Usage Data for All Users
  • POST Usage Data Repair for Active Users
  • POST Usage Data Repair for Group or System
  • POST Usage Data Repair for User
  • POST Usage Data Rollup
