GET User's Rating Plan

[Admin API Method]

Use this Admin API method to retrieve a user’s assigned rating plan (the plan that specifies how much to charge for service usage). This retrieves the full rating content, not just a rating plan ID.

Request Format

Syntax

GET /user/ratingPlan?userId=xxx&groupId=xxx[&region=xxx]

Query Parameters

userId

(Mandatory, string) User ID for user whose rating plan content is being retrieved.

 groupId

(Mandatory, string) Group ID for user whose rating plan content is being retrieved.

 region

(Optional, string) Service region with which the rating plan is associated. If not specified, default region is assumed.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
204 Rating Plan does not exist
400 Missing Required parameters : {userId, groupId}
400 Region {region} is not valid

Response Payload

RatingPlan JSON object.

 Example

In this example a user’s rating plan is successfully retrieved. The response body indicates that the user’s plan is the Default Rating Plan and it also indicates the substance of that plan in terms of how storage bytes and other dimensions of service usage are charged for.

# Request ->

GET http://192.168.2.16:18081/user/ratingPlan?userId=cprog&groupId=Dev
