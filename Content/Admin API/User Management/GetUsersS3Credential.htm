GET User's S3 Credential

[Admin API Method]

Use this Admin API method to retrieve the S3 credential object associated with a specified S3 access key. Along with the access key the credential object includes the secret key, creation date, and the credential’s status (active/inactive).

Request Format

Syntax

GET /user/credentials?accessKey=xxx

Query Parameters

accessKey

(Mandatory, string) The S3 access key for which associated credential information is being retrieved.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
204 No Data Found
400 Missing required parameters : {accessKey}

Response Payload

SecurityInfo JSON object

 Example

In this example a user’s S3 credential is successfully retrieved. Note that the user is not specified in the request — rather, the supplied access key uniquely identifies the credential.

# Request ->

GET http://192.168.2.16:18081/user/credentials?accessKey=0dfb7d248e850f8cbc3e

# Response ->

Status Code: 200 OK
Content-Length: 150
Content-Type: application/json
Date: Wed, 23 Sep 2015 13:54:23 GMT
{"active":true,"secretKey":"JitieOhfVdQ1sQ88yeGQyyLBSQr+aZbU0WreSs12","accessKey":
"0dfb7d248e850f8cbc3e","createDate":1440698072025,"expireDate":null}

For more information about SecurityInfo object attributes, see SecurityInfo.
