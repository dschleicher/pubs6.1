GET User's S3 Credentials List

[Admin API Method]

The HyperStore system allows a user to have multiple S3 security credentials. (The maximum number is configurable.) Use this Admin API method to retrieve all of a user’s existing S3 security credentials, including inactive credentials as well as active ones.

Request Format

Syntax

GET /user/credentials/list?userId=xxx&groupId=xxx

Query Parameters

userId

(Mandatory, string)  User ID for user whose S3 security credentials are being retrieved.

 groupId

(Mandatory, string) Group ID for user whose S3 security credentials are being retrieved.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
204 No Access Key found
400 Missing Required parameters : {userId, groupId}
400 User/Group does not exist

Response Payload

List<SecurityInfo> in JSON

 Example

In this example a user’s S3 credentials are successfully retrieved. The user has two credentials both of which are active.

# Request ->

GET http://192.168.2.16:18081/user/credentials/list?userId=cprog&groupId=Dev
