POST User's CMC Password

[Admin API Method]

Use this Admin API method to create or update a user’s Cloudian Management Console (CMC) password. Note that there is no PUT method for user’s CMC passwords — you use this POST method for creating a user’s initial password as well as for subsequently changing the password.

The HyperStore system encrypts users' CMC passwords before storing them.

Request Format

Syntax

POST /user/password?userId=xxx&groupId=xxx&password=xxx

Query Parameters

userId

(Mandatory, string) User ID for user whose CMC password is being posted.

 groupId

(Mandatory, string) Group ID for user whose CMC password is being posted.

 password

(Mandatory, string) User’s new or modified password. Maximum length is as configured by admin.user.password.length in mts.properties.erb (default max is 64 characters).

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 User does not exist
400 Missing Required parameters : {userId, groupId, password}
400 Exceeded max password length

Response Payload

None.

 Example

In this example a user’s CMC password is successfully updated.

# Request ->

POST http://192.168.2.16:18081/user/password?userId=kwring&groupId=Dev&password=nameofmycat

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 23 Sep 2015 14:53:48 GMT
