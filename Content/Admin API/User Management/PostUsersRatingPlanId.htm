POST User's Rating Plan ID

[Admin API Method]

Use this Admin API method to assign an existing rating plan to a user (for billing purposes).

To create a rating plan, use the PUT New Rating Plan method.

Request Format

Syntax

POST /user/ratingPlanId?userId=xxx&groupId=xxx&ratingPlanId=xxx[&region=xxx]

Query Parameters

userId

(Mandatory, string) User ID for user whose rating plan ID is being assigned.

 groupId

(Mandatory, string) Group ID for user whose rating plan ID is being assigned.

 ratingPlanId

(Mandatory, string) Rating plan ID that is being assigned to the user.

 region

(Optional, string) Service region with which the rating plan is associated. If no region is specified, the default region is used.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing Required parameters : {userId, groupId, ratingPlanId}
400 Region {region} is not valid

Response Payload

None.

 Example

In this example, the "Gold" rating plan is successfully assigned to a specified user.

# Request ->

POST http://192.168.2.16:18081/user/ratingPlanId?userId=cprog&groupId=Dev&ratingPlanId=Gold

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 23 Sep 2015 18:20:00 GMT
