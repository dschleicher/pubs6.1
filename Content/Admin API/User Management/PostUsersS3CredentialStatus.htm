POST User's S3 Credential Status

[Admin API Method]

Use this Admin API method to deactivate a user’s S3 credential, or to reactive a credential that is currently deactivated.

If you deactivate a credential, the user will not be able to use that credential to access the HyperStore S3 storage system. However, the credential will remain in the user’s credential list and can be reactivated at a later time. If you want to delete a credential, use theDELETE User's S3 Credential method instead.

Request Format

Syntax

POST /user/credentials/status?accessKey=xxx&isActive=xxx

Query Parameters

accessKey

(Mandatory, string) Access key, to identify the key pair that is being activated or deactivated.

 isActive

(Optional, boolean) Status to apply to the credentials — true for active or false for inactive. Defaults to false if this query parameter is not supplied in the request.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing required parameters : {accessKey}
400 Invalid Access Key

Response Payload

None.

 Example

In this example a user’s S3 credential is successfully deactivated. Note that the user is not specified in the request — rather, the supplied access key uniquely identifies the credential.

# Request ->

POST http://192.168.2.16:18081/user/credentials/status?accessKey=00f674da0a3b56a4e5b6&isActive=false

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 23 Sep 2015 14:31:31 GMT
