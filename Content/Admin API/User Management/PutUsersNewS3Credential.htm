PUT User's New S3 Credential

[Admin API Method]

Use this Admin API method to have the HyperStore system generate a new S3 security credential (access key and secret key) for a specified user.

The maximum number of credentials per user is set by the credentials.user.max property in mts.properties.erb. The default value is 5. Inactive credentials count toward this maximum as well as active credentials.

Request Format

Syntax

PUT /user/credentials?userId=xxx&groupId=xxx

Query Parameters

userId

(Mandatory, string) User ID for user whose S3 security credentials are being created.

 groupId

(Mandatory, string) Group ID for user whose S3 security credentials are being created.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Missing Required parameters : {userId, groupId}
400 User does not exist
403 Reached maximum number of credentials allowed

Response Payload

SecurityInfo JSON object.

 Example

In this example a new S3 credential is successfully created for a user.

# Request ->

PUT http://192.168.2.16:18081/user/credentials?userId=cprog&groupId=Dev

# Response ->

Status Code: 200 OK
Content-Length: 150
Content-Type: application/json
Date: Wed, 23 Sep 2015 14:10:24 GMT
{"active":true,"secretKey":"1m/PiKzK8V2I1TFVAZEWGUHs3LXyogeRB4mqfutU","accessKey":
"3fde4987d8987bc26ed9","createDate":1443017424340,"expireDate":null}

For more information about SecurityInfo object attributes, see SecurityInfo.
