POST Updated Whitelist Via URI Parameter

[Admin API Method]

Use this Admin API method to update the list of IP addresses and/or subnets that’s associated with the whitelist by passing the list membership as a URI parameter value.

This method overwrites the existing whitelist with the new list specified in the URI. You must include all of the IP addresses that you want on the updated list. For example, if there are four addresses on your current whitelist and you want to add two more addresses, list all six addresses in your request URI.

IP addresses and subnets are validated for correct syntax. The entire update request will be rejected if any address or subnet fails the syntax validation check.

The system does not enforce a limit on the number of IP addresses or subnets that you can include in the list.

You cannot use this method to "erase" a populated whitelist of IP addresses by submitting an empty list in the URI. To submit an empty list you must use the POST Updated Whitelist Via Request Object method instead.

Request Format

Syntax

POST /whitelist/list?whitelistId=xxx&list=xxx

Query Parameters

whitelistId

(Mandatory, string) Unique ID of the whitelist to update. In the current release this must be "Default-WL".

 list

(Mandatory, string) Comma-separated list of IP addresses or subnets. This list will overwrite the existing list associated with the whitelist ID specified by the "whitelistId" parameter. IP addresses can be IPv4 or IPv6 format. For subnets, only IPv4 format is supported in the current HyperStore release. IP addresses are validated for IPv4 or IPv6 syntax, and subnets are validated for CIDR syntax.

Request Payload

None.

 Response Format

Response Status

In addition to the Common Status Codes, this method may return these codes:

Status Code Description
400 Whitelist does not exist
400 Missing required attributes : {whitelistId, list}
400 Invalid IP Address or IPv4 Subnet CIDR: {value}

Response Payload

None.

 Example

In this example the whitelist is successfully updated. Note that the URI includes all IP addresses that are to be on the modified whitelist, not just the new additions.

# Request ->

POST http://192.168.2.16:18081/whitelist/list?whitelistId=Default-WL&list=192.168.2.10,192.168.2.11,
192.168.2.12,192.168.2.13,192.168.2.5,192.168.2.6

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 30 Sep 2015 13:35:21 GMT
