Restore an Auto-Tiered File

[CMC Interface]

If a bucket that you’re browsing has been configured for auto-tiering of objects older than a certain age, auto-tiered objects will display in the CMC's Objects view but will be marked with a special icon. Because these objects are currently stored in a remote tiered storage system (such as Amazon S3 or Amazon Glacier), you may not be able to directly download these objects. Whether they are directly downloadable or not depends on where they are stored — for example, objects in Amazon Glacier are never directly downloadable — and on the particular bucket lifecycle configuration that governed the auto-tiering of the objects.

You can try directly downloading an auto-tiered file by clicking on its file name. If direct download is not supported for the file, a response message will indicate that you need to temporarily restore a local copy of the file rather than directly downloading it.

To restore one or more auto-tiered files:

  1. Click the checkbox(es) to the left of the object name(s).
  2. In the "Restore" number of days field, enter the number of days for which you want the restored object(s) to be locally available in your HyperStore S3 service.
  3. Click the Restore button.

Note that restore does not happen instantly — it can take up to six hours before auto-tiered objects are restored to your local HyperStore S3 service (or up to nine hours for objects that have been auto-tiered to Amazon Glacier). In the interim, the file is marked with a special icon that indicates that it’s in queue for being restored. During this stage you cannot download the object.

After an object has been restored (as indicated by an icon), you can download it in the normal way.

Icons for Transitioned or Restored Files

In your file list display, the document icon to the left of file names will indicate if a file is transitioned, transitioning, restored, or restoring. The icons feature right-pointing or left-pointing red or blue arrows.

Icon Meaning
File is in the process of transitioning to a remote tiered storage system.
File has been transitioned to a remote tiered storage system.
A copy of the transitioned file is in the process of being restored to the local HyperStore S3 service. Download of the object is not yet supported.
A copy of the transitioned file has been restored to the local HyperStore S3 service. Download is supported.
