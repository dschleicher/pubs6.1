Auto-Tiering Credentials

[CMC Interface]

Path: Cluster → Cluster Config → Auto Tiering Credentials

Supported tasks:

  • Set default credentials for auto-tiering to Amazon

By default the HyperStore auto-tiering function is disabled and the Auto-Tiering Credentials page does not display in the CMC UI. If you want to enable the auto-tiering, go to the Auto-Tiering Settings section of the CMC's Configuration Settings page.

The Auto-Tiering Credentials page is for uploading default corporate AWS credentials in support of the HyperStore auto-tiering feature. For general information about the auto-tiering feature, see Auto-Tiering Feature Overview. For information specifically about AWS credentials and how they’re used by the auto-tiering feature, see Destination Accounts and Buckets.

To set your organization’s default AWS credentials for auto-tiering:

  1. Enter your AWS "Access Key" and "Secret Key".
  2. Maximum 128 characters for each key. Must be printable ASCII characters.
  3. Click Save.

If you subsequently view your credentials in this page, only the "Access Key" will be displayed — not the "Secret Key".

To delete a key pair, use the Delete button.
