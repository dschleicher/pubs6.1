Auto-Tiering Settings

[CMC Interface]

The settings in this section of the CMC’s Configuration Settings page configure the HyperStore system’s auto-tiering feature. After making any edits, click Save at the bottom of the page to dynamically apply the configuration change to the cluster.

For an overview of the HyperStore auto-tiering feature, see Auto-Tiering Feature Overview.

Enable Auto Tiering

This setting controls whether functionality for configuring buckets for auto-tiering will be enabled within the Bucket Lifecycle Rule section of the CMC’s Bucket Properties dialog.

Note that this setting controls specifically whether auto-tiering functionality is available through the CMC. Even if you have this setting disabled, auto-tiering is still supported through HyperStore extensions to the S3 API. This API functionality is available to S3 applications that have been customized to utilize the HyperStore API extensions, even if the auto-tiering functionality is disabled in the CMC.

Default = Disabled

 Auto Tiering URL

When auto-tiering is enabled in the CMC, this setting specifies the Amazon S3 region endpoint to tier to. This should be the Amazon S3 region endpoint URL most suitable for your HyperStore system’s location. For example:

  • In northern California, http://s3-us-west-1.amazonaws.com (or https://s3-us-west-1.amazonaws.com if you prefer to use HTTPS for auto-tiering)
  • In Japan, http://s3-ap-northeast-1.amazonaws.com (or https://s3-ap-northeast-1.amazonaws.com if you prefer to use HTTPS for auto-tiering)

The full list of Amazon S3 region endpoints is available at this Amazon Web Services webpage: http://docs.aws.amazon.com/general/latest/gr/rande.html#s3_region.

Once buckets have been configured for auto-tiering, you cannot change the S3 endpoint.

If you want outbound traffic from HyperStore to Amazon to pass through a forward proxy server, use a transparent proxy. The HyperStore system does not have settings for explicitly configuring such traffic to use a proxy.

Default = http://s3.amazonaws.com (this is for the U.S. East region)
