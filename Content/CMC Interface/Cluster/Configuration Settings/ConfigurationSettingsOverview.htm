Configuration Settings

[CMC Interface]

Path: Cluster → Cluster Config → Configuration Settings

Supported task: 

  • Edit HyperStore configuration settings

You can edit any of the following types of settings. Be sure to click Save at the bottom of the page to save your changes.

  • SMTP/Email Settings for Alerts/Notifications
  • SNMP Trap Destination Settings
  • System Settings
  • Usage Tracking Settings
  • Quality of Service Settings
  • Auto-Tiering Settings
  • S3 Request Restriction Settings
  • Auto-Repair Schedule Settings
  • LDAP Authentication Settings

When you change settings in the CMC’s Configuration Settings page the system applies your configuration changes dynamically — no service restart is required. At the same time, the system automatically updates the corresponding configuration file settings on the Puppet master node and triggers a Puppet sync-up so that the configuration file changes propagate to all your HyperStore nodes. This makes it so that your configuration changes persist across any future restarts of HyperStore services.
