hsstool clearsnapshot

[Command]

Use this hsstool command to delete old backup snapshots of a node. This operation deletes backup snapshots that were previously created by the hsstool snapshot operation.

There must be at least two snapshot increments in the snapshot directory in order to use the clearsnapshot command. See hsstool snapshot for information on how to list the existing snapshot increments.

Command Format

You can run the hsstool clearsnapshot command through the CMC UI:

Or on the command line:

[root]# /opt/cloudian/bin/hsstool -h <host> clearsnapshot <destination-dir> <older-than>

Parameters

This command supports these parameters:

-h <host> (called "Target Node" in CMC UI)

(Mandatory) Target host for which to delete snapshots.

 <destination-dir>

(Mandatory) Destination directory from which to delete snapshots. This must be a directory that has served as the destination directory for previous hsstool snapshot operations.

There must be at least two snapshot increments in this directory in order to use the clearsnapshot command. See hsstool snapshot for information on how to list the existing snapshot increments.

 <older-than>

(Mandatory) Delete only snapshots that are older than this age. You can specify an absolute date in format YYYY-MM-DD (for example 2014-05-10), or a time interval using m, D, W, M, Y to indicate minutes, days, weeks, months, or years respectively (for example, use "10D" if you want to delete snapshots older than 10 days).
