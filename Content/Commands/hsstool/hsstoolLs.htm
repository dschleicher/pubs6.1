hsstool ls

[Command]

This hsstool command returns a physical node’s list of HyperStore data mount points, the list of vNodes assigned to each mount point, and the current disk usage per mount point.

Command Format

You can run the hsstool ls command through the CMC UI:

Or on the command line:

[root]# /opt/cloudian/bin/hsstool [-h <host>] ls

Parameters

This command supports only this parameter:

-h <host> (called "Target Node" in CMC UI)

(Optional on command line; mandatory in CMC) Target host for which to retrieve each mount point’s vNode list and disk usage information. Defaults to local host if you do not supply -h <host> on the command line.

In a typical production environment, each host would have multiple disks that are mounted for HyperStore data storage. The ls command will return information for each such disk on the host.

For each disk on the target node, the ls command results display the following items:

Device name

Device name of the disk drive

 Mount point

Mount point of the device

 Total capacity of the disk, in KBs

Remaining available capacity of the disk, in KBs

 vNode list

For each HyperStore data mount point, the lower section of the ls command response lists all the vNodes assigned to that mount point. Each vNode is identified by its token number. Beside each vNode number is the HSFS directory corresponding to that vNode, in format /hsfs/<base62-encoded-vNode-token>. For more information on S3 storage directory structure see HyperStore Service and the HSFS.
