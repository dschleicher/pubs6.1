Checking Data Repair Status

Cluster-wide summary information about current automatic data repair activity is available in the CMC's Repair Status page. This includes information about any in-progress proactive repair or data rebalancing.

For detailed repair status information for a specific node, in the CMC go to Cluster → Nodes → Advanced and from the "Info" command group execute the hsstool opstatus command on that node.

To check whether there are currently any "hints" stored in the system (awaiting processing when a downed node comes back online), in the CMC go to Cluster → Nodes → Advanced and from the "Info" command group execute the hsstool hintsstatus command on any node.

To view the cluster-wide schedule for auto-repairs, in the CMC go to Cluster → Nodes → Advanced and from the "Info" command group execute the hsstool repairqueue command on any node.
