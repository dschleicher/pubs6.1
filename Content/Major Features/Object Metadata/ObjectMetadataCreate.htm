Creating User-Defined Object Metadata

To create user-defined object metadata in the HyperStore system you use the standard Amazon S3 API methods that implement this functionality. When an S3 client application submits a PUT Object request it can include one or more x-amz-meta-* headers that specify user-defined metadata. For example, x-amz-meta-project: apollo or x-amz-meta-topic: merger or x-amz-meta-author: uehara.

For HyperStore support of the PUT Object S3 API method, see PUT Object.

The Amazon S3 API method POST Object — for uploading objects via HTML forms — also allows for the specification of user-defined metadata (through x-amz-meta-* form fields), and Cloudian HyperStore supports this API method as well. For HyperStore support of this S3 API method, see POST Object.

In the current version of HyperStore, the Cloudian Management Console’s built-in S3 client does not support creating user-defined object metadata. So, to create user-defined object metadata you will need to use an S3 application other than the CMC.
