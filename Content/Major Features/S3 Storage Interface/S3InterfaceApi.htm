Using the S3 Storage Interface (S3 API)

HyperStore supports nearly all of the Amazon S3 REST API. HyperStore’s compliance with Amazon’s S3 API is covered in detail in the S3 API Support section of the Cloudian HyperStore documentation. To review which S3 API methods, parameters, headers, XML elements, and error codes HyperStore supports, see that documentation. That documentation also covers HyperStore extensions to the Amazon S3 API.

Because of HyperStore’s comprehensive compliance with the Amazon S3 API, you should be able to use most well-established S3 client applications with HyperStore. For feedback on particular S3 applications that you are considering using with HyperStore, consult with Cloudian Support.

By default the HyperStore S3 Service listens on port 80 for HTTP connections. The S3 Service does not support HTTPS (HTTP over TLS/SSL) by default, but you can set up HTTPS support for the S3 Service as described in SSL for S3 Service [XREF]. If you do so then the S3 Service will listen for HTTPS connections on port 443.

To check to see what is your HyperStore S3 Service endpoint (such as s3-region1.mycompany.com), in the CMC go to Cluster → Cluster Config → Cluster Information.
