Enabling AES-256

By default the HyperStore system does not support AES-256, the most secure form of the Advanced Encryption Standard. You must enable AES-256 if you want to do either of the following:

  • Use SSE in a manner compliant with the Amazon SSE specification
  • Use SSE-C

The HyperStore system supports the ability to test its regular SSE functionality (not SSE-C) using AES-128, rather than the more secure AES-256. If that’s all you want to do, you do not need to enable AES-256. HyperStore supports AES-128 by default.

To enable AES-256 in your HyperStore system, do the following:

  1. On every HyperStore node, download and install the Java Cryptography Extension (JCE) Unlimited Strength Jurisdiction Policy Files package for your JRE.

The cryptography extension package is available from the Java downloads site:

http://www.oracle.com/technetwork/java/javase/downloads/index.html

The cryptography extension packages are toward the bottom of the download page. Download and install the package that’s for your JRE version (which, as bundled with HyperStore, is JRE 7).

Installing this cryptograpy extension package will enable your JRE to support the AES-256 encryption standard. The default JRE supports only up to AES-128.

  1. On your Puppet master node, open this file in a text editor:
/etc/cloudian-<version>-puppet/modules/cloudians3/templates/mts.properties.erb

In mts.properties.erb, uncomment the following setting:

# cloudian.s3.serverside.encryption.keylength = 256

Then save and close the file.

  1. Still on your Puppet master node, launch the HyperStore install tool:
root# /<your-installation-staging-directory>/cloudianInstall.sh
  1. From the main menu select "Cluster Management" and then from the sub-menu that displays select "Push Configuration Settings to Cluster".
  2. After the Puppet run completes for all the Puppet agents, from the Cluster Management sub-menu select "Manage Services". Then from the Service Management sub-menu, restart your S3 Service to apply your configuration change.

AES-256 is now enabled in your HyperStore system.
