Using the CMC to Manage a Multi-Region System

Through the CMC you can monitor and manage an entire multi-region HyperStore system. If your HyperStore deployment is set up as a multi-region system, that will be reflected in these aspects of the CMC interface:

  • User Provisioning and Administration
    • When you Add a User, you will assign the user a rating plan (for billing purposes) for each region. You can assign the same rating plan in each region, or different rating plans in each region. The same is true when you Add a Group: when you set a group-level rating plan (which serves as the default rating plan for users within the group), you choose a rating plan for each region.
    • When you Set Quality of Service (QoS) for Users and Groups, you will assign a different set of limits for each region. As described in Multi-Region System Architecture, QoS limits are applied on a per-region basis.
  • S3 Data Storage and Access
    • When service users Add a Bucket, they can choose which region to create the bucket in.
  • Usage Reporting and Billing
    • When you Create a Usage Report for a user, a user group, or the system, you can choose the region for which to report usage. You can also generate reports that includes usage from all regions.
    • When you generate an Account Activity statement for billing a user, you generate a separate bill for each region.
  • HyperStore System Monitoring
    • When you check on Data Centers status, you choose which region you want to check on. Status information is available for every region.
    • When you check individual Node Status, you start by selecting a region, and then select a node within that region.
  • HyperStore System Management
    • When you perform management operations on individual nodes (such as cleanup or repair), you start by selecting a region, then select a node, then select the operation to perform.
    • When you are Adding a Node to the system, you specify the region in which to add it.
