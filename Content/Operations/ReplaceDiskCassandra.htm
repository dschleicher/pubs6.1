Replacing an OS/Cassandra Disk or SSD

Cloudian, Inc. recommends using RAID mirroring of the Cassandra and OS drives (and HyperStore Appliance machines are configured in this way). If a single drive fails, the other mirrored drive will continue to serve I/O. The recommended procedure for replacing a failed drive is:

  1. Remove the failed drive and replace it with a new drive of equal or greater capacity.
  2. Create a partition, format an ext4 file system, and mount it using the same mount point as was used for the drive you’re replacing.
  3. Use mdadm commands to add the new drive to your RAID set-up.

The new drive should then automatically sync up with the drive with which it is mirrored.
