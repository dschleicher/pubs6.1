Responding to a Cassandra Drive Nearing Capacity

If a Cassandra data drive (a disk or SSD storing the Cassandra database) is nearing capacity, the first two things to try are:

  • Use hsstool cleanup on the host node, using the allkeyspaces option. This will clear any Cassandra data that is no longer supposed to be on the node.
  • Selectively delete Cassandra data. To do this, consult with Cloudian Support.

If these measures do not free up sufficient space, the solution is to increase system capacity by adding one or more new nodes. For the procedure see Adding a Node. When you add a node, a portion of the Cassandra data on your existing nodes is copied to the new node and then deleted from the existing nodes — thereby freeing up space on the existing nodes.
