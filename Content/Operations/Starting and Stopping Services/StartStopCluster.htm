Start or Stop a Service on All Nodes

The interactive tool that you used to install the HyperStore system — cloudianInstall.sh — can also be used to manage HyperStore services. The tool enables you to start, restart, or start one or more services on all nodes at once.

The installation tool does not support managing a service on just one node within a multi-node cluster (for example, restarting the S3 Service on just one node within a cluster). To start, restart, or stop one or more services on just a single node, use the initialization scripts rather than the installation tool.

  1. On your Puppet master node, go to the staging directory in which the HyperStore install tool resides. This is the directory in which you unpacked the HyperStore release package and initiated the software installation.
[root]# cd <your-installation-staging-directory>
  1. Launch the install tool:
[root]# ./cloudianInstall.sh

This displays the top-level menu of options.

  1. Choose "Cluster Management", then from the sub-menu that displays choose "Manage Services". This displays the "Service Management" sub-menu:

  1. At the prompt, choose a service number from the menu. To manage all services choose option (0).
  2. At the prompt that appears after you make your service selection, choose a service command: start, stop, status, restart, or version.

The service command you choose will be applied to all nodes on which the service resides. For example, if you choose Cassandra and then "start", this will start Cassandra on all nodes on which it is installed. Likewise if you choose S3 and then "status", this will return the status of the S3 Service on each node on which it is installed. And if you choose "All services" and then "stop", this will stop all services on all nodes.

The "version" option is supported only for the S3 Service.

From the "Service Management" menu all you can do for the Puppet service is check its status. To stop or start the Puppet daemons, from the installer’s main menu choose "Advanced Options". From the advanced sub-menu that displays you can stop or start the Puppet daemons.
