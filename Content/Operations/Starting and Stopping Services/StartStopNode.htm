Start or Stop a Service on One Node

The HyperStore system includes initialization scripts that enable you to start, restart, or stop Cloudian HyperStore services on individual nodes. To run the initialization scripts you must have root privileges.

On the node that you're working with, the initialization script syntax is:

[root]# /etc/init.d/<service-name> [start|stop|restart|status|version|graceful]

The table below shows all Cloudian HyperStore services and their corresponding <service-name>.

Service <service-name>
S3 Service and Admin Service (these two services start and stop together) cloudian-s3
Cassandra Service cloudian-cassandra
HyperStore Service cloudian-hyperstore
Redis Credentials Service cloudian-redis-credentials
Redis QoS Service cloudian-redis-qos
Redis Monitor cloudian-redismon
Cloudian Management Console cloudian-cmc
Cloudian Monitoring Agent cloudian-agent
Dnsmasq cloudian-dnsmasq

The version option is supported only for the S3 Service (cloudian-s3).

The graceful option for service shut-down is supported only for the S3 Service and Admin Service (cloudian-s3), and the HyperStore Service (cloudian-hyperstore). The graceful option prevents any new incoming connections to the server but keeps existing connections open in order to let in-progress request processing complete before the server shuts down (within the bounds of relevant timeout checks). By contrast, the stop option and the restart option drop any open connections and abort any in-progress request processing.

If you are restarting multiple Redis nodes, make sure that each node is back up for at least one second before moving on to restart the next node.
