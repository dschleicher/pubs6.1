GET Amazon Tiering Credentials Source

[Admin API Method]

You can use this method to check which type of auto-tiering access credentials are being used by a particular HyperStore S3 bucket, or which type of auto-tiering access credentials are being used by your HyperStore system by default. The possible types are:

  • BUCKET — Bucket-specific Amazon S3 credentials that have been stored to the Redis Credentials database via the POST Amazon Tiering Credentials method.
  • SYSTEM — System-default Amazon S3 credentials that have been stored to the Redis Credentials database via the POST Amazon Tiering Credentials method.
  • LICENSE — Amazon S3 credentials stored in your HyperStore license file. These are for an Amazon account that Cloudian, Inc. creates specifically as an auto-tiering destination for your organization. For information about this auto-tiering option, consult with your Cloudian representative.
  • NONE — No Amazon S3 credentials are available in Redis or in your license file.

Note  This method does not return the actual Amazon S3 credentials — just the credentials source type. To retrieve actual credentials use the GET Amazon Tiering Credentials method.

Request Format

Syntax

GET /tiering/credentials/src[?bucketName=xxx]

Query Parameters

bucketName

(Optional, string) Specify the bucketName if you’re checking to see what type of auto-tiering credentials are used by a specific HyperStore S3 bucket. Omit this parameter if you want to see what type of auto-tiering credentials are used by the system by default.

Request Payload

None.

Response Format

Response Status

This method returns only the Common Response Status Codes

Response Payload

Tiering credential source type in text/plain.

Example

In this example, the Amazon tiering credential source type for the HyperStore bucket named "bucket1" is successfully retrieved. The response indicates that bucket-specific tiering credentials are being used for this bucket.

# Request ->

GET http://192.168.2.16:18081/tiering/credentials/src?bucketName=bucket1

# Response ->

Status Code: 200 OK
Content-Length: 6
Content-Type: text/plain
Date: Mon, 23 Nov 2015 12:03:08 GMT

BUCKET
