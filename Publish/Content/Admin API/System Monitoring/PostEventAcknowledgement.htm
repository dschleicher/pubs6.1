POST Event Acknowledgement

[Admin API Method]

Use this Admin API method to acknowledge one or more monitoring events for which the system has generated a notification. This changes the event notificationâ€™s status to "acknowleged".

Request Format

Syntax

POST /monitor/acknowledgeevents?nodeId=xxx[&region=xxx]

Query Parameters

nodeId

(Mandatory, string) Hostname of the node for which to acknowledge monitoring events.

region

(Optional, string) Service region in which the node located. If a region not specified in the request, the default region is assumed.

Request Payload

EventsAck JSON object.

Response Format

Response Status

In addition to the Common Response Status Codes, this method may return these codes:

Status Code Description
400 Missing required parameter : {nodeId, events}
400 Invalid region : {region}
400 Invalid JSON object

Response Payload

None.

Example

In this example, events associated with a node with hostname "appliance6" are successfully acknowledged. (This example is for acknowledging the set of events returned in the example from the GET Event List description.)

# Request ->

POST http://10.50.200.204:18081/monitor/acknowledgeevents?nodeId=appliance6

Content-Type: application/json

{"nodeId":"appliance6","eventTypes":["5","6","9|AuditDataCollector:populateNodeInfo(410)"]}

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Tue, 24 Nov 2015 16:44:00 GMT
