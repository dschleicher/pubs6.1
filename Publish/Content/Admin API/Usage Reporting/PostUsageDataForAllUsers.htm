POST Usage Data for All Users

[Admin API Method]

Use this Admin API method to post storage usage data (write stored bytes and stored object counts from the Redis QoS database over to the Cassanda "Raw" column family) for all users, not just active users. For background on posting storage usage data to Cassandra see POST Usage Data for Active Users.

Note  As long as cron.d on the node that hosts the HyperStore system cron jobs has been running continuously, there is no need to use this method, which is resource-intensive. The only time you might want to use this method is to recover from a cron service outage.

Note  This operation acts only on usage data in the local service region. In a multi-region system, the operation can be run locally in each region.

Request Format

Syntax

POST /usage/storageall

Query Parameters

None.

Request Payload

None.

Response Format

Response Status

This method returns only the Common Response Status Codes

Response Payload

None.

Example

In this example all users' service usage data from the Redis QoS database is successfully written to Cassandra where it will be available for further processing.

# Request ->

POST http://192.168.2.16:18081/usage/storageall

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 30 Sep 2015 13:56:36 GMT
