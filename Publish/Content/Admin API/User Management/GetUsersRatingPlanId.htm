GET User's Rating Plan ID

[Admin API Method]

Use this Admin API method to find out which rating plan is assigned to a specified user. This method returns only the rating plan ID — not the full rating plan object.

Request Format

Syntax

GET /user/ratingPlanId?userId=xxx&groupId=xxx[&region=xxx]

Query Parameters

userId

(Mandatory, string) User ID for user whose rating plan ID is being retrieved.

groupId

(Mandatory, string) Group ID for user whose rating plan ID is being retrieved.

region

(Optional, string)  Service region with which the rating plan is associated. If not specified, the default region is assumed.

Request Payload

None.

Response Format

Response Status

In addition to the Common Response Status Codes, this method may return these codes:

Status Code Description
204 Rating Plan does not exist
400 Missing Required parameters : {userId, groupId}
400 Region {region} is not valid

Response Payload

string (RatingPlanId)

Example

In this example a user’s rating plan ID is successfully retrieved.

# Request ->

GET http://192.168.2.16:18081/user/ratingPlanId?userId=cprog&groupId=Dev

# Response ->

Status Code: 200 OK
Content-Length: 10
Content-Type: text/plain
Date: Wed, 23 Sep 2015 17:52:24 GMT
Default-RP
