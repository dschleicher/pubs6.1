GET User's S3 Credentials List (Active)

[Admin API Method]

Use this Admin API method to retrieve only a user’s active S3 credetials. (To retrieve all of a user’s S3 credentials — inactive as well as active — use the GET User's S3 Credentials List method instead.)

Request Format

Syntax

GET /user/credentials/list/active?userId=xxx&groupId=xxx

Query Parameters

userId

(Mandatory, string) User ID for user whose active S3 security credentials are being retrieved

groupId

(Mandatory, string)  Group ID for user whose active S3 security credentials are being retrieved

Request Payload

None.

Response Format

Response Status

In addition to the Common Response Status Codes, this method may return these codes:

Status Code Description
204 No Access Key found
400 Missing Required parameters : {userId, groupId}
400 User/Group does not exist

Response Payload

List<GET User's S3 Credentials List> in JSON.

Example

In this example, a user’s active S3 credentials are successfully retrieved.

# Request ->

GET http://192.168.2.16:18081/user/credentials/list/active?userId=cprog&groupId=Dev
