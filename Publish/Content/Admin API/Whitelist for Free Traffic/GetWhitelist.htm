GET Whitelist

[Admin API Method]

Use this Admin API method to retrieve a specified whitelist.

Note  In the current HyperStore release only one whitelist is supported and its ID is "Default-WL".

Request Format

Syntax

GET /whitelist?whitelistId=xxx

Query Parameters

whitelistId

(Mandatory, string) Unique ID of the whitelist to retrieve. In the current HyperStore release, only one whitelist is supported and its ID is "Default-WL".

Request Payload

None.

Response Format

Response Status

In addition to the Common Response Status Codes, this method may return these codes:

Status Code Description
204 Whitelist does not exist
400 Missing required parameter : whitelistId

Response Payload

Whitelist JSON object.

Example

In this example the "Default-WL" whitelist is successfully retrieved. The results show that two IP addresses have been associated with the whitelist by system administrators. (When HyperStore is first installed, the Default-WL is empty. The list will be populated with IP addresses only if it has been edited by system administrators.)

# Request ->

http://192.168.2.16:18081/whitelist?whitelistId=Default-WL

# Response ->

Status Code: 200 OK
Content-Length: 115
Content-Type: application/json
Date: Wed, 30 Sep 2015 13:14:11 GMT
{"ratingPlanId":"Whitelist-RP","list":["192.168.2.10","192.168.2.11"],"name":"Default Whitelist",
"id":"Default-WL"}

For more information about Whitelist object attributes, see Whitelist.
