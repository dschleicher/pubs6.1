hsstool decommission

[Command]

Use this hsstool command to remove a live physical node from the storage cluster. When you decommission a node, the data associated with each vNode on the physical node is streamed to the vNode with the next-higher token, wherever it is in the storage cluster. Because there are many vNodes per physical node, and the vNodes are distributed randomly around the cluster, a decommissioned physical nodeâ€™s data is likely to be redistributed in a balanced way to the remaining nodes in the cluster.

The operational procedure during which you would use decommission (if the node is live) is:

  • Removing a Node

Please refer to that procedure for step-by-step instructions, including the proper use of decommission within the context of the procedure.

Command Format

You can run the hsstool decommission command through the CMC UI:

Or on the command line:

[root]# /opt/cloudian/bin/hsstool -h <host> decommission

Parameters

This command supports only this parameter:

-h <host> (called "Target Node" in CMC UI)

(Mandatory) Target host to decommission.
