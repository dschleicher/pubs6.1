hsstool info

[Command]

This hsstool command returns vNode information and data load information for a specified physical node within a storage cluster. The return includes a list of virtual nodes assigned to the physical node.

Command Format

You can run the hsstool info command through the CMC UI:

Or on the command line:

[root]# /opt/cloudian/bin/hsstool [-h <host>] info

Parameters

This command supports only this parameter:

-h <host> (called "Target Node" in CMC UI)

(Optional on command line; mandatory in CMC) Target host for which to retrieve token and load information. Defaults to local host if you do not supply -h <host> on the command line.

The info command response consists of the following items:

Cloudian

Cloudian HyperStore software version installed on the node.

Cloudian REPLICAS Load

The total volume of replicated S3 object data stored in the HyperScale File System on the node. This includes S3 objects for which the node serves as a secondary or tertiary replica as well as S3 objects for which the node is the primary replica.

This field also shows the total amount of disk space designated for S3 object storage on the node. The disk space is the total capacity for replica storage and erasure coded storage combined.

Cloudian EC Load

The volume of erasure coded S3 object data stored on the node. This field also shows the total amount of disk space designated for S3 object storage on the node. The disk space is the total capacity for replica storage and erasure coded storage combined.

Uptime (seconds)

The number of seconds that the HyperStore Service has been running since its last start.

Token

A list of storage tokens assigned to the node. These tokens, randomly generated from an integer token space ranging 0 to 2 127 -1, determine which S3 objects will be stored on the node. The multiple tokens assigned to each physical host are also known as "virtual nodes" or vNodes. By default each physical storage node is assigned 32 tokens. For more information on vNodes and S3 object allocation within the storage cluster, see How vNodes Work.

Cassandra

Cassandra software version installed on the node.

Cassandra Load

Cassandra storage load on the node. There will be some Cassandra load even if all S3 objects are stored in the HyperStore File System or the erasure coding file system. For example, Cassandra is used for storage of object metadata and service usage data, among other things.

Data Center

Data center in which the node resides.

Rack

Rack in which the node resides.
