Using the S3 Storage Interface (CMC)

The CMC’s Buckets & Objects section serves as a graphical S3 client with which users can interact with the HyperStore object store. With the CMC, users can do the following:

Work with Buckets

  • Add a Bucket
  • Set bucket properties
    • Set Custom S3 Permissions on a Bucket
    • Set "Canned" S3 Permissions on a Bucket
    • View a Bucket's Storage Policy Information
    • Configure a Bucket Lifecycle Policy for Object Auto-Tiering or Expiration
    • Configure a Bucket as a Static Website
    • Configure Cross-Region Replication for a Bucket
  • Delete a Bucket

Work with Objects

  • Create or Delete a "Folder"
  • Upload a File
  • Set file properties
    • Set Custom S3 Permissions on a File
    • Set "Canned" S3 Permissions on a File
    • Set Public URL Permissions on a File
  • Search for a File
  • Download a File
  • Delete a File
  • Restore an Auto-Tiered File

Note  The CMC system administrator role does not and cannot have its own S3 storage user account. However you can create a regular user account for yourself, and use that to access the data store. You can also manage other regular users' data on their behalf.

Note  When regular users log into the CMC, they do not see any of the CMC’s system management functions — only the Buckets & Objects section and other end user oriented features.
