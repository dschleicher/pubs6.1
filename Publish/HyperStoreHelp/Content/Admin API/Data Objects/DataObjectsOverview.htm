Admin API Data Objects

[Admin API Object]

This section describes data objects that are passed as request or response payloads via the HyperStore Admin API. The objects are documented in alphabetical order.

HyperStore Admin API response payloads are JSON encoded objects. Per the JSON specification (http://www.json.org), "An object is an unordered set of name/value pairs" (emphasis added). Accordingly, HyperStore Admin API clients should not expect that the ordering of attributes within a JSON-encoded response body will necessarily match the ordering of the attributes in the objectâ€™s documentation.

For POST or PUT requests that require a request payload, the request payloads must be JSON encoded as well. The documentation of these objects indicates whether each object attribute is mandatory or optional when submitted in request bodies. Note that:

  • Mandatory attributes cannot be assigned a null string nor an empty string value. They must be assigned a valid value as appropriate to the particular attribute.
  • Some optional attributes have a default value, as indicated in the documentation. For optional attributes that have a default value:
    • On PUTs the default is used if the attribute is empty in the PUT request body.
    • On POSTs the existing value in the system (from a prior PUT of the object) is retained if the attribute is empty in the POST request body.

Data Types

These are the standard data types used by the HyperStore Admin API:

Type Description
boolean Lower case literals true or false.
number A numeric value.
string Unicode characters. Note that within JSON objects (if you are constructing a JSON object to submit in a request body) strings must be enclosed with double-quotes (for example "host1").
null An empty value signified by the literal null.

The Admin API also utilizes certain custom data types that are built on the standard types. Custom types are described in the documentation of the data objects in which the custom types appear.
