QosLimitSettings

[Admin API Object]

The QosLimitSettings object is returned as the response payload for this Admin API operation:

  • GET QoS Limits

The QosLimitSettings object consists of the following attributes:

groupId

(string) Group ID. This will be either a specific group ID, or "ALL", or "*". For details of how this is used, see GET QoS Limits.

labelId

(string) This attribute is used by the CMC to display a title on a QoS configuration screen.

qosLimitList

(List<QosLimit>) List of <QosLimit> objects. There will be one QosLimit object for each of the eight QoS limit types.

userId

(string) User ID. This will be either a specific user ID, or "ALL", or "*". For details of how this is used, see GET QoS Limits.

Example

This is an example of a QosLimitSettings object. It indicates the QoS settings for a specific user, "cprog". All the QoS limit values are set to "-1" (meaning unlimited), so currently this user is not subject to any quality of service restrictions.

{"labelId":"qos.userQosOverrides.title","groupId":"Dev","userId":"cprog","qosLimitList":
[{"value":-1,"type":"STORAGE_QUOTA_KBYTES"},{"value":-1,"type":"REQUEST_RATE_LW"},{"value":-1,
"type":"REQUEST_RATE_LH"},{"value":-1,"type":"DATAKBYTES_IN_LW"},{"value":-1,"type":
"DATAKBYTES_IN_LH"},{"value":-1,"type":"DATAKBYTES_OUT_LW"},{"value":-1,"type":"DATAKBYTES_OUT_LH"},
{"value":-1,"type":"STORAGE_QUOTA_COUNT"}]}
