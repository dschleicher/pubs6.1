GET QoS Limits

[Admin API Method]

Use this Admin API method to retrieve QoS settings for a user or a group.

Request Format

Syntax

GET /qos/limits?userId=xxx&groupId=xxx[&region=xxx]

Query Parameters

userId

(Mandatory, string) User ID of the user whose QoS settings are being retrieved. Supported options are a specific user ID, "ALL", or "*", as described below the parameter list.

groupId

(Mandatory, string) Group ID of user whose QoS settings are being retrieved. Supported options are a specific group ID, "ALL", or "*", as described below the parameter list.

region

(Optional, string) Service region with which the QoS settings are associated. If not specified, the default region is assumed.

You can use the "userId" and "groupId" parameters to retrieve these categories of QoS settings:

  • User-level QoS for a specific user (userId=<userId>&groupId=<groupId>)
  • Default user-level QoS for a specific group (userId=ALL&groupId=<groupId>)
  • Default user-level QoS for the whole region (userId=ALL&groupId=*)
  • Group-level QoS for a specific group (userId=*&groupId=<groupId>)
  • Default group-level QoS for the whole region (userId=*&groupId=ALL)

The examples that follow show the request URI syntax for retrieving each category of QoS setting.

To retrieve current QoS settings for a specific user "kentaro" who belongs to group "cloud9":

GET /qos/limits?userId=kentaro&groupId=cloud9

To retrieve the default user-level QoS settings for users in group "cloud9":

GET /qos/limits?userId=ALL&groupId=cloud9

To retrieve the default user-level QoS settings for users in region "region1":

GET /qos/limits?userId=ALL&groupId=*

To retrieve the group-level QoS settings for group "cloud9":

GET /qos/limits?userId=*&groupId=cloud9

To retrieve the default group-level QoS settings for region "region1":

GET /qos/limits?userId=*&groupId=ALL

Request Payload

None.

Response Format

Response Status

In addition to the Common Response Status Codes, this method may return these codes:

Status Code Description
400 User does not exist
400 Missing required parameter : {userId, groupId}
400 Region {region} is invalid

Response Payload

QosLimitSettings JSON object.
