POST Updated Notification Rule

[Admin API Method]

Use this Admin API method to modify an existing notification rule.

Request Format

Syntax

POST /monitor/notificationrule

Query Parameters

None.

Request Payload

NotificationRule JSON object.

Response Format

Response Status

In addition to the Common Response Status Codes, this method may return these codes:

Status Code Description
400 Invalid JSON object
400 Notification rule Id does not exist : {ruleId}

Response Payload

None.

Example

In this example a notification rule is successfully updated. Note that in the request body, the "ruleId" attribute is specified to indicate which existing rule is being modified. This particular example modifies one of the notification rules that comes pre-created with the HyperStore system.

To get the ruleId for a rule that you want to modify, use the GET Notification Rules method. Once you have the ruleId you can then do the POST to modify the rule.

# Request ->

POST http://10.50.10.16:18081/monitor/notificationrule

Content-Type: application/json
{"email":"default","statId":"cpu","condition":">","conditionVal":"0.8","snmpTrap":false,
"ruleId":"12","enabled":true,"region":""}

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Fri, 04 Dec 2015 15:45:04 GMT

For more information about NotificationRule object attributes, see NotificationRule.
