POST Updated User

[Admin API Method]

Use this Admin API method to update profile information for an existing user. For example, you can change the user’s user type (such as promoting a regular user to become a group administrator), or suspend (or reactivate) a user, or update a user’s contact information.

If you suspend a user by changing the user’s status to inactive, the system automatically deactivates the user’s S3 security credentials. The user will not be able to access the S3 storage system or log into the CMC.

Note  To change a user’s rating plan, use the POST User's Rating Plan ID method. To change a user’s quality of service settings, use the POST QoS Limits method.

Request Format

Syntax

POST /user

Query Parameters

None.

Request Payload

UserInfo JSON object.

Response Format

Response Status

In addition to the Common Response Status Codes, this method may return these codes:

Status Code Description
400 User does not exist
400 Missing Required parameters : {userId, groupId, userType}
400 Invalid JSON object
400 Invalid User Name

Response Payload

None.

Example

In this example, a user’s profile is successfully updated (specifically, the user is promoted to become a group admin).

# Request ->

POST http://192.168.2.16:18081/user
Content-Type: application/json
{"address1":"134 Road St.","address2":"","city":"Downtown","zip":"90131","phone":"890-123-1324",
"website":"","active":"true","userType":"GroupAdmin","emailAddr":"kwring@enterprise.com",
"fullName":"Kassandra Wring","groupId":"Dev","userId":"kwring","state":"USA","country":"USA"}

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Tue, 22 Sep 2015 21:44:49 GMT

For more information about UserInfo object attributes, see UserInfo.
