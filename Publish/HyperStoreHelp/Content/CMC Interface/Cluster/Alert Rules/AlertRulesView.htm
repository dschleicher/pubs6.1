View Pre-Configured Alert Rules

[CMC Interface]

The HyperStore system comes with a set of pre-configured alert rules. The pre-configured alert rules are listed in the Rules section of the Alert Rules page. These pre-configured rules trigger node alerts if any of the following occur on a HyperStore node:

  • CPU utilization on a node exceeds 90%
  • Disk read/write error detected
  • Aggregate available disk space on a node falls below 10% of capacity (only disks used for Cassandra or HyperStore data directories are counted)
  • Available space on an individual disk falls below 15% of capacity
  • A data repair operation has completed.
  • A service goes down on a node on which the service is supposed to be running.
  • An ERROR level message has been written to a service log.

For the "service down" events, pre-configured alert rules are in place for each of the major services (Admin, Cassandra, HyperStore, Redis Credentials, Redis QoS, Redis Monitor, and S3). For the "ERROR in log" events, preconfigured rules are in place for all services except for the Redis Monitor.

If any of the system events covered by these pre-configured rules occur, an email is sent to your default system administrator email address(es) and an event entry displays in the Node Status page and also in the Alerts page.

Note   For more information about alerts, see How HyperStore Implements Alerts.

Optionally you can edit any of these pre-configured alert rules, or disable them, or delete them.
