View Cluster Information

[CMC Interface]

The CMC's Cluster Information page displays static information about your Cloudian HyperStore system. If you have a multi-region HyperStore deployment, a drop-down list at the top of the page lets you choose a region for which to view information.

The page shows the cluster information items below.

Version Information

Application Version

Your current HyperStore software version

License Information

Expire Date

Date on which your license expires.

IMPORTANT: The top of the console will display a warning message when your license expiration date is nearing. If your license expires, the message indicates how many grace period days you have until Cloudian, Inc. disables your HyperStore system.

Licensed Max Net Storage

Maximum allowed system-wide storage of S3 objects (in GB or TB), not counting replication or erasure coding overhead. For example if a 1MB S3 object is replicated three times in your system, or is protected by 4+2 erasure coding, it counts as only 1MB toward this limit.

Number of Stored Bytes

Current number of bytes of S3 objects stored in your system, not counting replication or erasure coding overhead. This is the number that counts toward your licensed storage limit.

Service Information

CMC Admin Server Host

Admin Service endpoint to which the CMC connects.

Name of your Cassandra cluster.

S3 Endpoints

HTTP and HTTPS endpoints for this S3 service region.

Redis Credentials Master Host

Host running the "master" instance of the Redis Credentials DB. If you have multiple regions, all regions use the same Redis Credentials master.

Redis Credentials Slave Host(s)

Host(s) running the "slave" instance(s) of the Redis Credentials DB. This is typically two nodes per data center. (It will be one node if you upgraded from a HyperStore version older than 6.0.)

Redis QoS Master Host

Host running the "master" instance of the Redis QoS DB. If you have multiple regions, each region has its own Redis QoS master.

Redis QoS Slave Host(s)

Host(s) running the "slave" instance(s) of the Redis QoS DB. This is typically one node per data center.

Redis Monitor Primary Host

Host running the primary instance of the Redis Monitor service.

Redis Monitor Backup Host

Host running the backup instance of the Redis Monitor service.

Admin Cronjob Primary Host

Host running the primary instance of the HyperStore system maintenance crontab.

Admin Cronjob Backup Host

Host with the backup instance of the HyperStore system maintenance crontab.

Internal NTP Host

Puppet Master Host

Host running the primary instance of the Puppet Master.

Puppet Master Backup Host

Host with the backup instance of the Puppet Master.

Installation Staging Directory on Puppet Master

On the Puppet Master host, the installation staging directory path. In this directory resides the cloudianInstall.sh script, which you can use for certain cluster management tasks such as pushing configuration file edits out to the cluster or restarting services.

Number of Datacenters

Number of datacenters in which you are running the HyperStore system.

Number of Hosts in Each Datacenter

Number of HyperStore hosts in each datacenter.
