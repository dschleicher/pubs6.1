View a Node's Services Status

[CMC Interface]

To check on the status of individual services on a node, in the CMC's Node Status page open the Services Status panel.

For each service, one of the following service status icons is displayed:

The service is up and running.
The service is down.

Note  The nodeâ€™s status is automatically checked and updated each minute, as indicated by the "Last Update" column.

Status is shown for the following services. Note that the Redis services will not necessarily be present on each node, if you have a multi-node system. For more detail about these services see HyperStore Services.

  • Admin -- Implements the Admin API that is invoked by the CMC and by HyperStore maintenance cron jobs.
  • Cassandra -- Manages access to the Cassandra database which stores various types of data in support of S3 service delivery. The Cassandra database may also store small S3 objects, depending on your system configuration.
  • HyperStore -- Manages access to the HyperStore File System (HSFS). Implements hsstool operations for managing the distributed S3 object store.
  • Redis Cred -- Manages access to the Redis Credentials database which stores S3 user credentials as well as additional data that supports S3 service delivery.
  • Redis QoS -- Manages access to the Redis QoS database which stores data required to implement quality of service controls.
  • Redis Mon -- Monitors Redis Credentials and Redis QoS cluster health and if needed implements automatic failover of the Redis master node role within each of the two Redis DBs
  • S3 -- Services S3 requests from client applications.
