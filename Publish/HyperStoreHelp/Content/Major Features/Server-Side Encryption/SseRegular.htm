Using Regular Server-Side Encryption (SSE)

You can use server-side encryption with system-generated keys either through the CMC or through a different S3 client application that invokes the HyperStore implementation of the S3 API.

Note  To use Amazon-compliant server-side encryption, you must first enable AES256 in your HyperStore system.

Using Regular SSE (CMC)

In the CMC’s Buckets & Objects page, when you are uploading a file, the interface displays a "Store Encrypted" checkbox. If you select this checkbox, the HyperStore system applies server-side encryption to the uploaded file, using a system-generated encryption key.

When you subsequently download the file through the Buckets & Objects page, the system automatically uses the same encryption key to decrypt the file.

Using Regular SSE (S3 API)

In compliance with the Amazon S3 REST API, the HyperStore system’s S3 API supports server-side encryption with system-generated keys by the inclusion of the x-amz-server-side-encryption request header in any of these operations on objects:

  • PUT Object
  • Initiate Multipart Upload
  • Upload Part
  • POST Object
  • PUT Object - Copy

HyperStore also supports the S3 API method PUT Bucket policy for setting bucket policies that mandate or forbid server-side encryption.
