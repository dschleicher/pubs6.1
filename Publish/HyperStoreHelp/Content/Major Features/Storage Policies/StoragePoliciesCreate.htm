Creating and Managing Storage Policies

You can create and manage storage policies through either the CMC or the Admin API.

Note  The system supports a configurable maximum number of storage policies (mts.properties: cloudian.protection.policy.max, default = 25). If this many policies already exist, you cannot create new policies until you either delete unused policies or increase the configurable maximum.

Creating and Managing Storage Policies (CMC)

In the CMC you can do everything you need to do in regard to creating and managing storage policies in one location: the Storage Policies page. In that page you can:

  • Add a Storage Policy
  • Edit a Storage Policy
  • Designate a Default Storage Policy
  • Disable a Storage Policy
  • Delete a Storage Policy

At all times you must have one and only one default storage policy defined in in each of your HyperStore service regions. The default policy is the one that will be applied when users create new buckets without specifying a policy.

IMPORTANT: When a pre-5.2 version of HyperStore is upgraded to 5.2 or later, the upgrade process automatically creates a storage policy named "DEFAULT_<region-name>". This policy is configured with the data distribution scheme (replication factor or EC "k"+"m" values) and user data read/write consistency requirements that the pre-5.2 system had been using. This "DEFAULT_<region-name>" will be your default storage policy until you designate some other policy as the default. You can disable this policy (after designating a different default policy) so that it cannot be used for newly created buckets, but you cannot delete this policy.

Creating and Managing Storage Policies (Admin API)

The Admin API supports several calls for creating and managing storage policies:

  • Create a storage policy: PUT /bppolicy
  • Edit a storage policy: POST /bppolicy
  • Retrieve a list of storage policies: GET /bppolicy/listpolicy
  • Retrieve a storage policy: GET /bppolicy
  • Disable a storage policy: DELETE /bpppolicy/disable
  • Re-enable a disabled storage policy: PUT /bppolicy/enable
  • Delete a storage policy: DELETE /bpppolicy

Note  The "bp" in these calls stands for bucket protection. "Bucket protection policy" is another term for storage policy.
