Security Credentials

Path: Drop-down menu under your user name (at top right) → Security Credentials

Supported tasks:

  • Change your Console password
  • Manage your S3 access keys
  • Check the S3 service endpoint

Note  Some of these functions may or may not display in the Console, depending on configuration choices made by service administrators.

Change Your Console Password

Note  If the storage system is configured to use LDAP authentication, the CMC’s Change Password function will not be available. Instead, passwords should be controlled through your organization’s LDAP system.

  1. In the "New Password" field, enter a new password that you will use to log into the CMC. Maximum supported password length is 64 characters.
  2. In the "Confirm Password" field, enter the new password again.
  3. Click Change Password.

Manage Your S3 Access Keys

S3 access keys are used by S3 client applications (such as the Console) to authenticate requests submitted to the HyperStore S3 service.

  • To create a new S3 data access key, in the S3 Access Credentials dialog click Create New Key. A new access key ID then displays in the access key list.
  • To view the secret access key that corresponds to an access key ID, to the right of the access key ID click View Secret Key. A secret key display dialog appears which enables you to view your secret key and to copy it using <Ctrl>-c if you want to. Note that the OK and Cancel buttons have no effect other than to close the secret key display dialog.

Note: If you view your secret key(s) multiple times during one Console login session, your browser may display a "Prevent this page from creating additional dialogs" checkbox that appears as if it’s part of the Console UI. Do not select this checkbox. If you select this checkbox and then click OK you will no longer be able to view your secret access keys during your current Console login session.

  • To activate or deactivate an access key, to the right of the displayed access key ID click Activate or Inactivate.
  • To delete an access key, to the far right of the displayed access key ID click Delete. You will be asked to confirm that you want to delete the key.

Check the S3 Service Endpoint

The Service Information panel displays the S3 endpoints (HTTP and HTTPS) for the storage service. An endpoint is a URL that client applications use to connect to a web-based service. S3 client applications use these S3 endpoints to when submitting requests to the storage service.
