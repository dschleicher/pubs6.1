Configure a Bucket as a Static Website

You can configure a bucket as a website in order to make all of its content available to public users using regular web browsers. Only static websites are supported. A static website is a website that serves the same content to all visitors (no content personalization based on cookies or other mechanisms). A static site does not use server-side scripting.

Note  Public access to buckets configured as static websites is through HTTP not HTTPS. The static website feature does not support HTTPS (SSL).

To configure a bucket as a static website, select the Static Website Hosting tab of the CMC's bucket properties interface.

  1. Select the "Enable Website Hosting" checkbox.
  2. Complete the "Index Document" field with the name of the file that should be served when site visitors access the root URL of your site. This file is typically named "index.html".
  3. Complete the "Error Document" field with the name of the file that should be served when site visitors trigger an HTTP 4xx error (such as by requesting an object that does not exist). You might name this file "404.html", for instance.
  4. Click Save.

Note  If you have not yet done so, you must upload both the index document file and the error document file into the root level of your bucket, and make sure their names correspond exactly to those you specified in the Static Website Hosting interface.

When you configure a bucket as a static website, the system automatically applies a bucket access policy that allows the public to read objects in the bucket. If you later disable the static website configuration for the bucket (by unchecking the "Enable Website Hosting" checkbox in the Static Website Hosting interface), the system removes this public read access policy for the bucket.

For a bucket configured as a static website, the root URL will be:

http://<bucketName>.<regionalS3WebsiteEndpoint>
