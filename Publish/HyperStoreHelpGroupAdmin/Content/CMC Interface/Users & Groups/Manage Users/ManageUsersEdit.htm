Edit a User

  1. Use the CMC's Manage Users page to Retrieve a User or List of Users.
  1. In the "Actions" column for the user that you want to edit, click Edit. This displays a panel for editing the user’s attributes.

  1. Make your desired changes to the user’s attributes:
Active User
  • Active User checkbox checked — User is active. User has access to the storage service and the CMC.
  • Active User checkbox unchecked — User is suspended. User will be denied access to the storage service and the CMC. However, the user will remain in the system and their objects will remain in storage.

Note: If you want to remove a user entirely and delete their stored objects, then you must delete the user.

User Type
  • User -- Regular user of the storage service. He or she will be able to use the CMC to create storage buckets, upload and download objects, display reports on their service usage, and manage their service access credentials.
  • Group Admin -- An administrator of your group. He or she will be able to use the CMC to perform administrative functions for the group, such as adding users or setting user-level QoS profiles. A group admin has a storage service account and can upload their own objects to storage. A group admin can also manage stored objects on behalf of particular users within the group.
Canonical ID

The system automatically generates a unique canonical ID for every user. This ID cannot be edited.

Password

The user’s password for logging into the CMC.

The Password and Confirm Password fields will be blank. If you want to leave the user’s password unchanged, leave these fields blank. If you want to change the user’s password, complete these fields.

Maximum allowed length is 64 characters.

Note for systems using LDAP authentication

Do not edit a user’s password in the CMC if your HyperStore system is configured to use LDAP authentication. When LDAP is used, the CMC authenticates users against their password stored in your LDAP server. The password that’s stored in the CMC is simply a system-generated random string that serves as a placeholder within the HyperStore user profile schema. When LDAP integration is enabled, this CMC-stored password string is not actually used for user authentication.

Full name and contact information

User's full name (maximum 64 characters) and contact information.

  1. Click Save.
