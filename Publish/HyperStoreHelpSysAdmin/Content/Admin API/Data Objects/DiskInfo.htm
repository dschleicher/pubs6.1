DiskInfo

[Admin API Object]

A list of DiskInfo objects — one for each disk on the node — is nested within the MonitorNodeInfo object.

The DiskInfo object consists of the following attributes:

deviceName

(string) Disk drive device name.

diskAvailKb

(string) Total remaining free space on the disk in number of KBs. In calculating this value, a disk’s "reserved-blocks-percentage" (the portion of the disk space that’s reserved for privileged processes) is considered to be used space. By default in Linux systems the configurable "reserved-blocks-percentage" for a file system is 5% of disk capacity.

diskTotalKb

(string) Total capacity of the disk in number of KBs.

mountPoint

(string) File system mount point for the disk.

status

(EDiskStatus) Disk status string. One of "OK", "ERROR", "DISABLED", or "DISABLEDNOMOVE".

For description of these disk statuses, see View a Node's Disk Status.

storageUse

(EStorageType) List of storage type strings, indicating what type of data is being stored on the disk. One or more of:

  • "CASSANDRA" — System metadata and S3 object metadata in Cassandra.
  • "HS" — Replicated S3 object data.
  • "EC" — Erasure coded S3 object data.
  • "NOTAVAIL" — Storage usage information cannot be retrieved.

Example

This is an example of a DiskInfo object. The example is an excerpt from within a parent MonitorNodeInfo object. It indicates status information for a single disk on the node.

{"mountPoint":"/","diskAvailKb":"61243876","diskTotalKb":"67049088",
"deviceName":"/dev/mapper/vg0-root","storageUse":["CASSANDRA","HS"],"status":"OK"}
