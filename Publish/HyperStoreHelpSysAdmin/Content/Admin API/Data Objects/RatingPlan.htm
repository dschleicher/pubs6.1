RatingPlan

[Admin API Object]

The RatingPlan object is returned as the response payload for these Admin API operations:

  • GET Rating Plan
  • GET User's Rating Plan

The RatingPlan object is required as the request payload for these Admin API operations:

  • POST Updated Rating Plan
  • PUT New Rating Plan

The RatingPlan object consists of the attributes below. Each attribute description indicates whether the attribute is mandatory or optional when this object is transmitted as a request payload.

currency

(Optional, string) An international currency code (such as "USD", "JPY", or "EUR").

id

(Mandatory, string) Unique rating plan ID.

mapRules

(Optional, Map<string, RuleClass>) Map of rating rules per service dimension. The string is the service dimension acronym and is one of "SB" (storage bytes), "BI" (bytes in), "BO" (bytes out), "HG" (HTTP GETs), "HP" (HTTP PUTs), or "HD" (HTTP DELETEs). For each service dimension there is a corresponding RuleClass object that expresses the rating rules for that service dimension.

name

(Optional, string) Name of rating plan.

Example

This is an example of a RatingPlan object. In this rating plan HTTP DELETEs are unbilled, and every other service dimension has either a one-tier or two-tier pricing structure, in U.S. dollars.

{"currency":"USD","mapRules":{"HG":{"rules":[{"second":"0.02","first":"10"},{"second":"0.01",
"first":"0"}],"ruleclassType":"HTTP_GET"},"HD":{"rules":[{"second":"0","first":"0"}],
"ruleclassType":"HTTP_DELETE"},"BI":{"rules":[{"second":"0.20","first":"1"},{"second":"0.10",
"first":"0"}],"ruleclassType":"BYTES_IN"},"BO":{"rules":[{"second":"0.20","first":"1"},{"second":
"0.10","first":"0"}],"ruleclassType":"BYTES_OUT"},"HP":{"rules":[{"second":"0.02","first":"0"}],
"ruleclassType":"HTTP_PUT"},"SB":{"rules":[{"second":"0.14","first":"1"},{"second":"0.12","first":
"5"},{"second":"0.10","first":"0"}],"ruleclassType":"STORAGE_BYTE"}},"name":"Default Rating Plan",
"id":"Default-RP"}
