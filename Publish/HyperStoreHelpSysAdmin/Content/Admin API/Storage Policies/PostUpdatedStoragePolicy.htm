POST Updated Storage Policy

[Admin API Method]

Use this Admin API method to revise an existing storage policy. Only the following policy attributes are allowed to be revised:

  • Policy name and description
  • Read/write consistency levels required by the policy
  • Groups for which the policy is available
  • Whether the policy is the default policy for the region

Note  You cannot use this method to re-enable a policy that has been disabled. For information on re-enabling a disabled policy, see the POST (Re-Enable) Storage Policy method.

Request Format

Syntax

POST /bppolicy

Query Parameters

None.

Request Payload

Revised BucketProtectionPolicy JSON object.
