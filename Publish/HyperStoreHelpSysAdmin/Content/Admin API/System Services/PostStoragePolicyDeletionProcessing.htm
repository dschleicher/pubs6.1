POST Storage Policy Deletion Processing

[Admin API Method]

Use this Admin API method to process any pending storage policy deletion jobs.

System operators can initiate the deletion of an unused storage policy (a storage policy that is not assigned to any buckets) through the DELETE Storage Policy method, or through the CMC. This operator action makes the policy immediately unavailable to service users. However, the full process of deleting the storage policy from the system is not completed until the method described below is run.

Note  This method is invoked once a day by a HyperStore Storage Policy Deletion Processing cron job. It runs only in your default service region.

Request Format

Syntax

POST /system/processProtectionPolicy

Query Parameters

None.

Request Payload

None.

Response Format

Response Status

This method returns only the Common Response Status Codes

Response Payload

None.

Example

In this example any pending storage policy deletion jobs are successfully completed.

# Request ->

POST http://10.50.10.16:18081/system/processProtectionPolicy

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Sun, 03 Jan 2016 13:01:38 GMT
