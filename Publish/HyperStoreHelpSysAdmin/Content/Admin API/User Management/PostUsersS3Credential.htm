POST User's S3 Credential

[Admin API Method]

Use this method to have the HyperStore system store an S3 credential (access key and corresponding secret key) that you supply in the request. The user can then use this credential to access the HyperStore S3 storage system.

Note  To have the system generate a credential (rather than your supplying the credential in the request), use the PUT User's New S3 Credential method instead.

Request Format

Syntax

POST /user/credentials?userId=xxx&groupId=xxx&accessKey=xxx&secretKey=xxx

Query Parameters

userId

(Mandatory, string) User ID for user whose S3 security credentials are being stored.

groupId

(Mandatory, string) Group ID for user whose S3 security credentials are being stored.

accessKey

(Mandatory, string) User’s S3 access key.

secretKey

(Mandatory, string) User’s S3 secret key.

Request Payload

None.

Response Format

Response Status

In addition to the Common Response Status Codes, this method may return these codes:

Status Code Description
400 Missing Required parameters : {userId, groupId, accessKey, secretKey}
400 User does not exist
403 Reached maximum number of credentials allowed
409 Access Key already exists

Response Payload

None.

Example

In this example an access key and corresponding secret key have been successfully submitted to the system so the specified user can use that credential to access the S3 storage service.

# Request ->

POST http://192.168.2.16:18081/user/credentials?userId=kwring&groupId=Dev&accessKey=
00f674da0a3b56a4e5b6&secretKey=2ZxHvh2eLZe9cztZXEKHPGsvS0BMoeX9tnxWTax3

# Response ->

Status Code: 200 OK
Content-Length: 0
Date: Wed, 23 Sep 2015 14:22:16 GMT
