Set "Canned" S3 Permissions on a File

[CMC Interface]

"Canned" S3 permissions (also known as "canned ACLs") are pre-defined sets of S3 permissions that you can assign to a file. To set canned permissions on a file, in the CMC select the Object Canned ACL tab in the file properties interface.

  1. From the drop-down list, choose the canned ACL to assign to the file. You can only choose one. Options are:
Private

File owner has full access rights and no one else has access rights. This is the default.

Public Read

File owner has full access rights and the Public grantee group has read access.

Authenticated Read

File owner has full access rights and the Cloudian User group has read access.

Bucket Owner Read

File owner has full access rights and the bucket owner has read access.

Bucket Owner Full Control

File owner and bucket owner both have full access rights.

  1. Click Save.

Note  A file can have only one canned ACL attached to it at a time. If you apply one canned ACL to a file, then later apply a different canned ACL, the first canned ACL will be removed.
