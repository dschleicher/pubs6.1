LDAP Authentication Settings

[CMC Interface]

The settings in this section of the CMC’s Configuration Settings page enable and configure HyperStore integration with your LDAP system. After making any edits, click Save at the bottom of the page to dynamically apply the configuration change to the cluster.

Note  For complete information on this feature see Integrating with an LDAP System.

Enable LDAP Authentication

Whether to enable or disable the CMC’s LDAP authentication feature.

Note that if you have existing users that have been created in the CMC, and then you enable LDAP authentication, those pre-existing users will no longer be able to log into the CMC. For more detail see Integrating with an LDAP System.

Default = disabled

LDAP Server URL

The URL that the CMC should use to access your LDAP Server when authenticating users.

Note that if you use ldaps (LDAP secured by SSL/TLS), your LDAP server must use a CA-verified certificate not a self-signed certificate. HyperStore does not support connecting to an LDAP server that’s using a self-signed SSL certificate.

Default = ldap://hostname:389/o=MyOrg

User DN Template

Set this to the DN (Distinguished Name) template by which users are uniquely identified in your LDAP system. This template specification must include the DN attribute "uid" set to equal the exact literal string "{userId}" and the DN attribute "ou" set to equal the exact literal string "{groupId}". For example:

uid={userId}, ou={groupId}, dc=my-company, dc=com

Or for another example:

uid={userId}, ou={groupId}, dc=my-organization, dc=org

Note that you should set the "uid" and "ou" attributes to the required string, but customize the "dc" attributes to values appropriate for your particular organization.

Default = uid={userId}, ou={groupId}, dc=company, dc=com
