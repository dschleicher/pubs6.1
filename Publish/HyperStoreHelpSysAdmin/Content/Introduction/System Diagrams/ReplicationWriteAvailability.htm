Availability for Replicated Object Writes

[Diagram]

The scenarios in this section presume that 3X replication is being used to protect objects in a six node HyperStore cluster, and that only one data center is being used. When you configure storage policies for replication in a single DC you can choose one of three supported consistency level (CL) requirements for S3 object data writes: ALL, QUORUM, or ONE. You can separately configure CL requirements for S3 object metadata writes, choosing from ALL or QUORUM.

The diagrams that follow show S3 write availability under various cluster health scenarios and configured consistency level requirements. (For information about how the system selects write endpoints for each object and its metadata, see How vNodes Work).

Note  In cases where the overall S3 write request succeeds despite the failure of one or more replica writes, the missing replicas are subsequently created either through hinted handoff or proactive repair (thereby achieving "eventual consistency").
