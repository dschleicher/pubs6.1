Starting and Stopping Services

You have two options for starting, restarting, and stopping Cloudian HyperStore services:

  • Start or Stop a Service on All Nodes (using the HyperStore installer)
  • Start or Stop a Service on One Node (using an init.d script)

By default all of the HyperStore services on a host are configured to automatically start when the host is booted up (the configuration setting is common.csv: service_starts_on_boot, which defaults to "true" [XREF]). This includes dnsmasq if it was included in your HyperStore software installation.

Note  ntpd is configured to automatically start on host boot-up. However, it’s recommended that after booting a HyperStore host, you verify that ntpd is running (which you can do with the ntpq -p command — if ntpd is running this command will return a list of connected time servers).
